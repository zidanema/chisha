### 2. POST /api/sandbox/sessions/{sid}/eat
- body: `{rec_rank: int, mock_recommend?: bool}`
- 流程 (同步部分):
  1. 读 sessions/{sid}/last_recs.json → 取 rec_rank-1
  2. 调 `/api/accept` 内部函数 (持有 last_recs.recommend_session_id),传 rec_id
  3. history append {idx, state:"eat", dish, session_id}
  4. meal_to_trace[idx] = recommend_session_id
  5. `sandbox.advance_meal(sid=sid)` 推进
  6. 清 last_recs.json (下顿要重 /recs)
  7. 启动 BackgroundTask: build_decision_async(sid, meal_idx)
  8. 返 `{job_id, status:"running", new_meal_idx}`

- BackgroundTask `build_decision_async`:
  1. 读 prev long_term_prefs.json snapshot (落到 sessions/{sid}/_prev_lt.json,eat 前快照)
  2. 触发 L1 抽取 (复用 /api/long_term_prefs/refresh 内部函数),`record_l1_extraction(..., trigger="meal_eat", sid=sid)`
  3. 读 new long_term_prefs.json
  4. 调 `build_decision(...)` 派生 Decision
  5. 写 sessions/{sid}/decisions/{idx}.json
  6. 更新 job 表 status="done"

### 3. POST /api/sandbox/sessions/{sid}/skip
- 无 body
- 流程: history append {idx, state:"skip"} + advance_meal + 清 last_recs
- decision: 同步直接落 sessions/{sid}/decisions/{idx}.json (skip 不触发 L1 抽取)
- 返 `{new_meal_idx, decision}`

### 4. POST /api/sandbox/sessions/{sid}/swap
- body: `{exclude_ids?: list[str], mock_recommend?: bool}`
- 重 `recommend_meal(exclude=exclude_ids ∪ 当前 last_recs.ids)` → 覆盖 last_recs
- 不推进 meal_idx
- 返 `{currentRecs: Rec[]}`

### 5. POST /api/sandbox/sessions/{sid}/refine
- body: `{text: str, mock_recommend?: bool}`
- 调内部 `refine_session(session_id=last_recs.recommend_session_id, user_input=text, ...)` (复用 web_api.api_refine 已有逻辑)
- 覆盖 last_recs.json
- 返 `{currentRecs: Rec[], activeRules: {refine: [{label: text, sinceRound: ...}], blacklist: []}}`
- **B panel 单 round 文案**: refine 只在当前 last_recs/round 内有效,advance_meal 后清空。所以 activeRules.refine 由 `last_recs.applied_refine` 派生 (落 last_recs.json 里),meal advance 时自动消失。

### 7. Adapter (chisha/sandbox_adapter.py 新建)

`format_v2_candidate → Rec` view-model 转换:
- rank, name, venue, dishes, price 直接映射
- l1Hits: 从 candidate 内 l1_hits 或 l1_trace 提 top-2
- l2 = score (0-1), l3 = rerank_score (0-100), boost = round(l3 - l2*100)
- intent = refine_intent.summary_text (refine 路径有) 或 "context-fit" (recommend 路径)
- explore = is_explore
- conflict = 从 hard_filter_events 命中当前 rec 派生 (refine exclude_keywords)
- meta {eta, dist, protein, oil} = 已有字段
- why = reasoning

### 8. ?mock_recommend=1 query

支持的端点: /recs, /eat, /swap, /refine。
- query parsing: web_api 端点 wrapper 检测 query,绕过 recommend_meal/refine_session
- 返 mock 5 条 (从 sbxMocks 数据形状) + mock recommend_session_id (uuid)
- mock 路径不写 trace v3,只走 sandbox state 流程

## Why

- 推荐链路真实跑过 sandbox (复用 recommend_meal/refine_session,不重写)
- async 让 14 顿 × LLM 20s 不阻塞前端
- mock_recommend 让 chrome-devtools-mcp 自测可重复跑不烧 token
