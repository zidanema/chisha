**依赖 S-04。S-06a/b/c/07 全依赖。**

参考: `docs/design_briefs/2026-05-19-sandbox-lab.md` §多 session 落盘改造 + §per-meal 时钟。

## What

`chisha/sandbox.py` 从单全局态 → 多 session 目录结构,migration 老数据,state.json 扩 per-meal 字段。

### 目录结构 (sandbox 启用后)

```
logs/sandbox/
├── _meta.json                          # {"schema_version": 2, "migrated_at": iso}
└── sessions/
    ├── _default/                       # D-077 兼容 (旧用例落这里)
    │   ├── state.json
    │   ├── meal_log.jsonl
    │   ├── recommend_log.jsonl
    │   ├── recommend_trace/
    │   ├── long_term_prefs.json
    │   ├── profile.yaml
    │   ├── feedback/store.json
    │   ├── sessions/                   # 老 D-077 的 session_id-level dir (recommend session,非 sandbox session)
    │   ├── decisions/                  # 本任务新建 (S-06b 写)
    │   └── meal_to_trace.json          # 本任务新建 (S-06c 写)
    ├── _legacy/                        # migration 把老 logs/sandbox/*.* 挪进来
    │   └── (与 _default 同 layout, 但只读)
    └── {sid}/                          # 用户新建 sandbox sessions
        └── (同上)
```

### state.json schema (per-session)

```json
{
  "sid": "_default",
  "name": "默认 session",
  "seed": 42,
  "profile_label": "profile@v2",
  "origin": "real_snapshot",
  "enabled": true,
  "current_date": "2026-05-15",
  "day_index": 1,
  "current_meal_idx": 0,
  "total_meals": 14,
  "history": [],
  "branch_from": null,
  "branch_from_meal_idx": null,
  "started_at_real": "2026-05-19T15:00:00+00:00",
  "started_at_virtual": "2026-05-15",
  "last_l1_extraction": null

```

`last_l1_extraction` 内层字段:
```json
{
  "at": "iso",
  "status": "ok|failed|pending|skipped",
  "based_on_meals": 4,
  "error": null,
  "trigger": "meal_eat|day_advance"
}
```
(Codex must-fix #4 trigger 字段)

### chisha/sandbox.py API 扩展

新增:
- `list_sessions(root=None) -> list[dict]` — 扫 `logs/sandbox/sessions/*/state.json` 全部
- `create_session(*, name, days=7, seed, profile_label, origin, sid=None, root=None, copy_real_data=False) -> dict` — 新建 session,落 state.json,返完整 state
- `delete_session(sid, root=None) -> dict` — `shutil.rmtree(sessions/{sid}/)`
- `migrate_legacy(root=None) -> dict` — 检测老 `logs/sandbox/state.json` 是否存在 → 挪到 `sessions/_legacy/`,落 `_meta.json`。**幂等可重跑** (_meta.json 已存在则 noop)。

兼容:
- `init(start_date=None, *, sid="_default", root=None, copy_real_data=False)` — 老调用走 `_default` session,内部走 `create_session(name="默认 session", sid="_default", ...)`
- `advance(days=1, *, sid=None, root=None)` — sid=None → ctx → "_default"
- `reset(*, sid=None, root=None)` — sid=None → 删 `logs/sandbox/` 全部 (D-077 兼容); sid="_default" → 同上; sid="xyz" → 只删 `sessions/xyz/`
- `disable(*, sid=None, root=None)` — sid 同上,enabled=false。**全局 `is_enabled()` 仍然检测 `logs/sandbox/_meta.json` 存在** (整个沙盒目录在则 enabled,不依赖 per-session enabled)。
  - 注:per-session `enabled` 仅用于 done/running status 显示,**`is_enabled()` 是全局 sandbox 模式开关**
- `state(*, sid=None, root=None)` — 同上
- `current_date(*, sid=None, root=None)` / `current_datetime` / `current_datetime_utc` — 同上,默认 _default

## Why

- 满足前端"多 session 切换"需求 (handoff §session 选择器)
- branch/rollback 落地基础 (sessions/{sid} 整目录拷贝/裁剪)
- D-077 兼容: 旧用例不改

## Done When

- `chisha/sandbox.py` 落上述 API,_meta.json + migrate_legacy 幂等
- `uv run pytest tests/ -q` 全绿
- migration 自测:
  - 删 `logs/sandbox/` → init() → 自动建 sessions/_default/
  - 在 `logs/sandbox/state.json`(老 layout) 放一份 → migrate_legacy → 挪到 sessions/_legacy/
- baseline_l2_snapshot 0 diff (本任务不动推荐链路)
