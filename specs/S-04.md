## What

让 backend 永远显式 sid,不靠全局 active 标记。两个 tab 并发 eat 不串。

### 1. 新建 `chisha/sandbox_context.py`

```python
"""Sandbox per-request session context.

ContextVar-based: 每个 FastAPI 请求 wrapper 用 `with set_sandbox_session(sid):` 注入,
data_root / clock 透明读 ctx。绝不全局可写。

Codex must-fix #1: 包装调用 (eat 内部串 /recommend) 不靠 active.json 全局切换,
两 tab 并发 eat 不串。
"""
from contextlib import contextmanager
from contextvars import ContextVar

_current_sid: ContextVar[str | None] = ContextVar("sandbox_session_id", default=None)

@contextmanager
def set_sandbox_session(sid: str | None):
    token = _current_sid.set(sid)
    try:
        yield
    finally:
        _current_sid.reset(token)

def current_sandbox_session() -> str | None:
    return _current_sid.get()
```

### 2. 改 `chisha/data_root.py`

所有 `*_path()` 函数加 `session_id: str | None = None` 参,默认从 `sandbox_context.current_sandbox_session()` 读。

行为矩阵:
| sandbox 启用 | session_id 显式 | ctx sid | 实际路径 |
|---|---|---|---|
| False | * | * | prod 路径 (旧行为) |
| True | "abc" | * | `logs/sandbox/sessions/abc/{rel}` |
| True | None | "xyz" | `logs/sandbox/sessions/xyz/{rel}` |
| True | None | None | `logs/sandbox/sessions/_default/{rel}` (D-077 兼容) |

实现 helper:
```python
def _maybe_sandbox(root, rel_in_sandbox, rel_in_prod, session_id=None):
    if not sandbox.is_enabled(root):
        return root / rel_in_prod
    sid = session_id or current_sandbox_session() or "_default"
    return root / "logs" / "sandbox" / "sessions" / sid / rel_in_sandbox
```

所有 7 个落盘点 + recommend_trace_dir + profile_path 都改:
- meal_log_path, sessions_dir, feedback_store_path, recommend_log_path, feedback_history_path, long_term_prefs_path, recommend_trace_dir, profile_path

### 3. 改 `chisha/clock.py`

`clock.today(root)` 内部:
```python
def today(root=None):
    if sandbox.is_enabled(root):
        d = sandbox.current_date(root, session_id=current_sandbox_session())
        if d:
            return d
    return real_today()
```

`sandbox.current_date(root, session_id=None)` (本任务先扩签名,实现在 S-05):
