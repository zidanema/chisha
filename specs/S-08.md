### 4. apps/sandbox-lab/src/hooks/useSandbox.ts 改

S-03 的 useSandbox 改成:
- 启动时 ping `/api/sandbox/sessions`:
  - 成功 → backendOnline=true, 初始拉 sessions 列表 + 默认 active 第一个 → GET /sessions/{sid} → setStore
  - 失败 → backendOnline=false, fallback 到 sbxMocks
- 顶栏 pill: backendOnline ? "backend · :8765" : "mock · offline" (橙色,见 debug-ui ThemeSwitcher 旁边的 .pill.live 样式)
- actions 全部改成 fetch:
  - handleEat(rec) → POST /eat → 收 job_id → 启动 polling (1.5s 间隔,直到 done) → done 后 GET /sessions/{sid} 拉新 FullSnapshot
  - handleSkip / handleSwap / handleRefine 同理
  - selectCell 不动 (本地状态)
  - handleRollback → POST /rollback → 拉新 FullSnapshot
  - handleBranch → POST /branch → push 新 SessionMeta + activate
- 用 ?mock_recommend=1 query (本地存 localStorage 或 ?dev=1 query 触发)

### 5. App.tsx 接 polling 状态

- eat/skip/swap/refine 期间显 skeleton (style-guide §2): 5 RecCard 区域显 5 个 skeleton 卡; banner 区显 "处理中..."
- polling 失败 (job 表 error) → toast + 不推进 state
