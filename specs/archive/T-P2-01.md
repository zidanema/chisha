# T-P2-01 · reference 块 resolver + 比较器

参考: `docs/design_briefs/2026-05-18-refine-v2-faithful-refine.md` §10 P2-01。

## What
1. 实现 `last_meal_id` 解析层：解析 refine 文本里的"昨天那顿"/"上次那家"/"上周三午饭"等相对引用，从 `trace_store` 里找到对应历史 session 的 final 5 道。
2. 实现"更清淡"/"相似但换家" 比较器：
   - `lighter` → 在历史 meal 基础上加 oil_level 下限约束 + 蔬菜占比上限约束
   - `similar_but_different_venue` → 保留 cuisine + ingredient 主体，restaurant_id 不同
   - 其他 relation 可扩展
3. 解析结果进入 `RefineIntent.reference` 块（T-P1a-03 已加 schema slot 但未消费 — 本任务消费）。
4. trace 可见 reference 解析结果（reference_meal_id resolved → 哪 5 道 + 比较器选项）。

## Why
- 用户用"相对表达"是对话记忆预期的临界点：搜索框 → 私人助理的产品体感升级。
- T-P1a-03 已经解析出 reference slot 但 L1/L2 不消费 — 本任务把它真正用上。
- 5 类典型 reference 表达打通后，"听懂但做不到"清单减少一大块。

## Done When
- 5 类典型 reference 表达可执行：「比昨天清淡」「和上次那家差不多但换一家」「比上周三午饭便宜」「和那次外婆来吃的那顿一样」「不要像那次那样」（最后这条是 negative，可降级提示）。
- trace 可见 reference 解析结果（resolved_meal_id + 哪 5 道 + 应用的 relation 转换）。
- 5+ 失败 case（找不到 last meal / 模糊引用 / 时间超出 trace 范围）100% 走降级路径不崩。
- `uv run pytest tests/ -q` 全绿。
- 空输入 + 不含 reference 的 refine 路径 baseline_l2_snapshot 0 diff。

## Affected files (预估)
- `chisha/reference_resolver.py`（新建）
- `chisha/refine_intent.py` 或 extractor：消费 reference slot
- `chisha/recall.py` 或新建 `chisha/refine_comparator.py`：lighter / similar_but_different_venue 比较器
- `chisha/trace_store.py`：读历史 session
- 测试: `tests/test_reference_resolver.py`

## 红线
- trace_store 读取必须用现有 API（不要绕过 D-079 frozen prefs 等保护）。
- reference 解析失败必须降级到空 refine 模式或纯 redirect 模式，绝不崩。

## 不做
- 不做 `avoid_pattern`（"不要像那次那样"）的真实执行 — brief §12 明确 V2/V3。可降级到提示用户「这个表达我还做不到，按你说的菜系推了一组」。
- 不引入新的 LLM 调用（resolver 走 trace_store 查询即可）。
