# T-P1b-01 · 顶部 always-on 状态条

参考: `docs/design_briefs/2026-05-18-refine-v2-faithful-refine.md` §8。

## What
在 `apps/web/` 顶部加 always-on 状态条：
1. 空输入时：展示当前 profile + methodology 简述（如「低油 · 高蛋白 · 蔬菜≥50%」）。
2. refine 触发 L0-A 拒绝时：显式告知「检测到你对 X 过敏，已忽略 refine 里的 Y。若已治愈去 profile 改」。
3. refine 触发 L0-B 拒绝时：类似 A，文案换成「按你的素食 / 清真设定，已忽略 refine 里的 Y」。
4. refine 触发 L0-C 解除时：显式告知「破戒模式 · 今晚一次性放开 · 不影响周报」。
5. 数据源：T-P1a-01 落地的 `hard_filter_event` trace 流。

## Why
- 让用户感知"系统在为我做什么"。
- Faithful Refine 第一原则的体感载体 — 用户看不见的执行 = 不存在的执行。
- §8 强调 narrative + 状态条是"信任放大器"，必须在硬契约下沉 L1 之后才能上（否则给不靠谱的系统装无敌嘴皮）。

## Done When
- 启动 `uv run python -m chisha.debug_server` + `cd apps/web && npm run dev`，顶部能看到状态条。
- 空输入：能看到当前 profile 摘要。
- 注入 refine "想吃花生酱面" + profile 标花生过敏：状态条显示 A 类提示。
- 注入 refine "今晚就放纵给我推炒饭"：状态条显示 C 类解除提示。
- chrome-devtools-mcp 自驱验证（强制，见 CLAUDE.md「前端自测」）：navigate / golden path / console_messages 无 error / network_requests 无 4xx-5xx / 截图。
- `cd apps/web && npm run test` (vitest) + `npm run typecheck` 全绿（如有）。

## Affected files (预估)
- `apps/web/src/components/StatusBar.tsx`（新建）
- `apps/web/src/App.tsx`（挂上）
- `apps/web/src/api/*` 或 hooks：消费 trace 流 / hard_filter 事件
- `chisha/web_api.py`：可能需要新端点暴露 hard_filter_event（如 `/api/session/{sid}/active-constraints`）
- 后端可能扩 `chisha/api.py`

## 红线
- 改 apps/web/src/** 后必须 chrome-devtools-mcp 自驱验证（CLAUDE.md 强制）。
- 数据真实：所有提示必须来自 trace `hard_filter_event` 真事件，禁止前端硬编码假数据 fallback。
- 不动用户视图主交互（D-052~D-055 + D-060/D-066/D-067 style-guide 锁定不可重设计）。

## 不做
- 不做 narrative 卡片（T-P1b-02）。
- 不做调试台 SPA（apps/debug-ui）— 那是另一条链路。
- 不引入新视觉设计 token，复用现有 style-guide。
