# S-09 · trace 跳转到 debug-ui

**依赖 S-08。**

## What

把 sandbox-lab 的"打开 trace"按钮 (timeline op-bar / ReviewCard / DPanel) 改成跳 debug-ui :5174,query 带 trace_id + round_id。

### 1. 前端: apps/sandbox-lab/

`onOpenTrace(meal: Meal | null)`:
- 从 `meal_to_trace[meal.idx]` (从 FullSnapshot 拿) 取 trace_session_id
- meal=null 时跳当前顿 (从 last_recs.recommend_session_id)
- `window.open(\`http://127.0.0.1:5174/?trace=${tsid}&round=R1\`, "_blank", "noopener")`

### 2. 后端: 确保 GET /sessions/{sid} 返 meal_to_trace

S-06a FullSnapshot shape 加 `mealToTrace: Record<string, string>` (idx → tsid)。本任务确认 S-06a/c 已写,如果没有补回去。

### 3. 后端: GET /api/sandbox/sessions/{sid}/meal/{idx}/trace 端点 (可选 helper)

替代前端从 meal_to_trace 派生,也可以直接 GET 一次:
- 返 `{trace_session_id: str, round_id: "R1"}`
- 404 if no trace for that meal (skip)

本任务可以不实现这个端点,直接前端从 FullSnapshot.mealToTrace 派生,更省事。

### 4. apps/debug-ui/ 改: 接受 URL query 自动选 trace

- `apps/debug-ui/src/App.tsx` 或 `useWaTrace.ts` 加:
  ```ts
  useEffect(() => {
    const params = new URLSearchParams(window.location.search);
    const trace = params.get('trace');
    const round = params.get('round') || 'R1';
    if (trace) {
      setActiveTraceId(trace);
      setActiveRound(round);
    }
  }, []);  // 仅启动一次
  ```
- 注意: 不破现有 chisha:tbCollapsed / chisha:theme localStorage 行为
- 如果 trace 不存在 → 仍然走 mock fallback (与 backend offline 同路径)

### 5. tests + 自测

- pytest: 后端不动 (除非补 /meal/{idx}/trace 端点),则 tests/test_sandbox_meal_trace_endpoint.py
- 前端: `npm run typecheck` 全绿
- chrome-devtools-mcp e2e:
  - 起 :8765 + :5174 (debug-ui) + :5175 (sandbox-lab)
  - 在 :5175 跑完一顿 (mock_recommend=1) 或拿现有 trace 数据 → 点 timeline op-bar "打开 trace" → 新 tab 跳 :5174/?trace={tsid}&round=R1 → 验证 trace 在 :5174 自动选中
  - 同样验 DPanel "打开 trace" 按钮 + ReviewCard "打开完整 trace" 按钮

## Why

handoff §关键交互 §timeline op-bar 要求"打开 trace 跳外部 trace 平台"。chisha 现成 debug-ui 就是 trace 浏览平台,直接接。

## Done When

- 3 处 onOpenTrace 按钮 (timeline op-bar / DPanel "打开 trace" / ReviewCard "打开完整 trace") 全部走 window.open
- debug-ui 启动时接 ?trace=&round= query 自动选 trace
- chrome-devtools-mcp 验证 3 处按钮跳转都正确

## Affected files

- 改: `apps/sandbox-lab/src/{App.tsx, components/Timeline.tsx, components/ReviewCard.tsx, components/panels/DPanel.tsx}` (绑 onOpenTrace)
- 改: `apps/debug-ui/src/App.tsx` 或 `src/hooks/useWaTrace.ts` (加 query 解析)
- 不动: 后端 (除非补可选端点)

## 不做

- 不内嵌 trace 视图到 sandbox-lab 内 (handoff 明确"跳外部 trace 平台")
- 不改 debug-ui 其他行为
