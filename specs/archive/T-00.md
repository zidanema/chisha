# T-00 · trace event schema + intent schema 版本号

**前置基础设施。P1a/P1b 全部依赖。先做，0.5 天。**

参考: `docs/design_briefs/2026-05-18-refine-v2-faithful-refine.md` §9 前置基础设施。

## What
1. 在 `chisha/trace_store.py` 加 `hard_filter_event` 结构。事件 schema 至少含:
   - `event_type: "hard_filter"`
   - `category: "L0_A_medical" | "L0_B_identity" | "L0_C_health" | "methodology"`
   - `rule: str`（如 "蔬菜占比≥50%"、"oil_level<=medium"）
   - `dropped_count: int`
   - `kept_count: int`
   - `refine_override: bool`（C 类是否被 refine 解除）
   - `timestamp: float`
2. 在 `chisha/refine_intent.py`（或 schemas 相关文件）的 intent 结构里加 `schema_version: str` 字段，初值 `"1.0"`。
3. 在 `docs/CONTRACTS.md` 顶部写入 Faithful Refine 第一原则 + L0 三分判定表（A 医学/B 身份/C 健康，A/B 永不可破，C refine 可破）。

## Why
- P1a/P1b 都依赖。先建占位让两者真正并行。
- schema_version 用于后续 P1a-03 多 slot 升级时无侵入扩展（旧 trace 仍能回放）。
- Faithful Refine 是新宪法，必须先入 CONTRACTS 让后续 PR 有可对照的约束。

## Done When
- `trace_store` schema 加好 + 单测覆盖向后兼容（旧 trace 不带新事件仍能 load）。
- 真跑一次推荐 dry_run，trace 落盘看到新字段（即使值为空数组）。
- `refine_intent` 结构带 `schema_version`，旧调用方不破。
- `docs/CONTRACTS.md` 顶部能搜到 "Faithful Refine" 和 "L0 三分"。
- `uv run pytest tests/ -q` 全绿。
- `uv run python -m scripts.baseline_l2_snapshot --out-dir tmp/baseline_traces_after && uv run python -m scripts.compare_traces`：0 diff（trace schema 加字段但旧路径行为不变）。

## Affected files (预估)
- `chisha/trace_store.py`
- `chisha/refine_intent.py`（或 schemas.py）
- `docs/CONTRACTS.md`
- 测试: `tests/test_trace_store.py`（新增或扩）

- 不实际写入任何 hard_filter event（那是 T-P1a-01 的事）。本任务只建 schema 占位 + 验证向后兼容。
- 不改任何 L1/L2/L3 逻辑。
