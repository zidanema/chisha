# S-03 · 交互骨架 mock (eat/skip/refine/timeline/rollback/branch)

**依赖 S-01, S-02。S-08 联调将此 mock 替换为真后端。**

参考: `docs/design_briefs/2026-05-19-sandbox-lab.md`,交互语义源 `chidiansha-sandbox/project/sandbox-lab/HANDOFF_PROMPT.md` §关键交互 + §验收清单。

## What

在 S-02 视觉基础上,把 HANDOFF_PROMPT.md §验收清单 11 条交互全部跑通,**数据全走本地 React state + mock**,不接后端。

### 1. `apps/sandbox-lab/src/hooks/useSandbox.ts` (新建,本任务核心)

返 store + actions:
- `sessions: SessionMeta[], activeSessionId, setActiveSessionId`
- `clock: {idx, day, slot, total}` (从 idx 派生 day=idx//2+1, slot=idx%2===0?"午":"晚")
- `history: Meal[], currentRecs: Rec[], lastDecision: Decision | null`
- `selectedCellIdx: number | null` (回顾模式选中)
- `transientBanners, dismissedBannerIds`
- actions:
  - `handleEat(rec)` — history 加一条 eat + advance 1 idx + 重 mock recs + 清 transient banner
  - `handleSkip()` — history 加 skip + advance + D panel 文案"跳过未学习"
  - `handleSwap()` — recs 重排 + 加 swap transient banner
  - `handleRefine(text)` — 加 refine_applied transient banner (单 round 文案,无 TTL),mock 把 recs 重排
  - `selectCell(idx)` — 切回顾模式 + D panel 切到那顿 + banner 切快照
  - `clearSelection()` — 退出回顾
  - `handleRollback(idx)` — history.slice(0, idx) + clock 回 + selectedCell 清
  - `handleBranch(fromIdx, name)` — sessions push 新条目 + activeSessionId 切到新 sid + history 裁到 fromIdx
  - `dismissBanner(id)`

派生 banners 规则 (设计 brief + handoff §banner 语义):
- `refine_conflict` — 派生自 `currentRecs.some(r => r.conflict)`,**不存 state**
- transient (swap/refine_applied) — 存 state,clock 推进时 setTransientBanners([])
- 回顾模式 — selectedCell !== null && !== clock.idx 时,banner stack 切快照 (review-mode header + 该顿当时的 conflict/refine flags)

### 2. `apps/sandbox-lab/src/hooks/useKeyboard.ts` (新建)

- Esc 触发: clearSelection + 关 SummaryDrawer + 关 ConfirmModal + 关 RefineModal
- 不实现其他快捷键 (Cmd+K 等留 v2)

### 3. App.tsx 接 hook

- session 下拉切换 → `setActiveSessionId` → 重渲染 (mock 数据可以每个 session 一份固定)
- "就这个 →" → `handleEat(rec)`
- "跳过" → `handleSkip()`
- "换一组" / preset chip → `handleSwap()` / `handleRefine(text)`
- timeline 点格 → `selectCell(idx)`
- op-bar:打开 trace 按钮 (本任务 mock window.alert 即可),回滚 → ConfirmModal kind="rollback",分支 → ConfirmModal kind="branch"
- ConfirmModal onConfirm:回滚走 handleRollback,分支走 handleBranch
- 主区切换:`selectedCellIdx !== null && !== clock.idx` 时渲 ReviewCard,否则 DecisionArea

### 4. `apps/sandbox-lab/src/mocks/sbxMocks.ts` 扩

- 提供 `freshRecsForMeal(mealIdx, sessionId) → Rec[]`,让 eat/skip/swap 后能拿到新 mock recs (避免一直显示同一组)
- `buildDecisionMock(rec, clock) → Decision`,模拟 S-06b decision diff 派生器的 mock 版本
