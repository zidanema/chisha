# S-07 · rollback / branch 端点 + 文件原子裁剪

**依赖 S-06c。S-08 调用。**

参考: `docs/design_briefs/2026-05-19-sandbox-lab.md` §rollback/branch 完整裁剪。Codex gotcha #1 + #3。

## What

### 1. POST /api/sandbox/sessions/{sid}/rollback {meal_idx}

裁掉 meal_idx 及之后的所有状态,回到 meal_idx 这一顿的"准备决策"状态。

流程 (in order,**全部 atomic — tmp file + rename**):

1. **Validate**: 0 <= meal_idx < current_meal_idx (否则 400)
2. **读 meal_to_trace.json** 取 idx >= meal_idx 的所有 trace_session_ids → 收集为 trash_tsids
3. **删 trace 文件**: 对每个 tsid 删 `sessions/{sid}/recommend_trace/{tsid}.json`
4. **裁 meal_to_trace.json**: 只保留 idx < meal_idx,atomic rewrite
5. **删 decisions/{i}.json** for i >= meal_idx
6. **裁 meal_log.jsonl**: 按 timestamp / 行内 meal_idx 字段过滤,tmp file + rename (按 sandbox 推进顺序写的,可以按行数裁——精确按行数 = sum(history[:meal_idx].state == "eat"))
7. **裁 recommend_log.jsonl**: 同上,按 trace_session_id 在 trash_tsids 内的行剔除
8. **重置 long_term_prefs.json**: 清空 (设为 `{}`),让下次 eat 重抽 L1
9. **删 last_recs.json**: 用户决策前要重新 /recs
10. **清 active jobs**: 把 _JOB_TABLE 内 sid 相关 entry 删 (in-memory)
11. **更新 state.json**:
    - `current_meal_idx = meal_idx`
    - `history = history.slice(0, meal_idx)`
    - `current_date` 回算 (meal_idx_to_slot derive)
    - `day_index = meal_idx // 2 + 1`
    - `last_l1_extraction = None` (与清空 long_term_prefs 一致)
12. 返新 FullSnapshot

错误处理: 任何一步失败必须把已删的部分 rollback (用 try/finally + 备份目录),或者**用单步原子 commit**: 把所有改动先写到 `sessions/{sid}/.rollback_tmp/`,完整成功后整体 rename → 替换。

### 2. POST /api/sandbox/sessions/{sid}/branch {from_meal_idx, name}

新 sid = 拷贝 src + 裁到 from_meal_idx。

流程:
1. **Validate**: 0 <= from_meal_idx < src.current_meal_idx
2. 生成新 sid: `sandbox_{ts}_{hex(4)}`
3. `shutil.copytree(sessions/{src}/, sessions/{new}/)` (全量拷贝,含 trace / decisions / meal_log / recommend_log / long_term_prefs / profile / feedback / state.json)
4. 在 new 上跑 rollback 逻辑裁到 from_meal_idx (复用 #1)
5. 改 new 的 state.json:
   - `sid = new_sid`, `name = name`, `branch_from = src_sid`, `branch_from_meal_idx = from_meal_idx`
   - `started_at_real = now`
6. 返新 SessionMeta

### 3. tests (`tests/test_sandbox_rollback_branch.py`)

- 用 mock_recommend=1 跑 4 顿 → rollback to idx=2 → 验:
  - state.history.length === 2
  - state.current_meal_idx === 2
  - meal_to_trace 只剩 {0,1}
  - decisions/2.json 删, decisions/3.json 删
  - recommend_trace 只剩 idx<2 的文件
  - long_term_prefs.json === {}
  - last_l1_extraction === None
- rollback 后再 eat → 验:
  - 新 trace 写入,不读到老 idx=2 的 tsid (Codex gotcha #1 验证点)
- branch from idx=2,name="alt" → 验:
  - new sid 在 list_sessions 返
  - new state.history.length === 2
  - new long_term_prefs.json === {}
  - new branch_from === src_sid
  - src.history.length === 4 (不变)
- 原子性: monkey-patch shutil.copy 在第 N 步 raise → 验 state.json 仍是 pre-rollback 完整版本

## Why

- timeline op-bar "回滚到这里" / "从这里分支" 完整动线
- 数据隔离: rollback 不污染未来 trace, branch 不继承未来记忆 (Codex gotcha #3)

## Done When

- 2 端点上线,Pydantic body 校验
- 7+ 单测全绿
- baseline_l2_snapshot 0 diff
- 手测: 在 mock_recommend=1 下跑完 4 顿 → rollback to 2 → 再 eat → 不读到老 trace

## Affected files

- 改: `chisha/web_api.py` (加 2 endpoint + atomic helper)
- 改: `chisha/sandbox.py` 可能加 `rollback_session(sid, meal_idx)` `branch_session(src_sid, from_meal_idx, name)` 业务函数 (端点 thin wrapper)
- 新建: `tests/test_sandbox_rollback_branch.py`
- 不动: 前端 / 推荐链路

## 不做

- 前端调用 (S-08)
- 跨 PR 的 cross-session branching (新 sid 在 new 目录,src 不动)
