# S-06a · sessions CRUD 端点

**依赖 S-05。S-06c/07/08 调用。**

参考: `docs/design_briefs/2026-05-19-sandbox-lab.md`,handoff §endpoints。

## What

提供 sandbox session 的 list / create / detail (read-only) 端点。**不实现 eat/skip/swap/refine 等写端点** (那是 S-06c)。

### 端点 (chisha/web_api.py,全部 _require_localhost)

#### GET /api/sandbox/sessions
- 返 `list[SessionMeta]`
- 内部调 `sandbox.list_sessions(root=ROOT)`,逐项转 SessionMeta:
  ```ts
  {
    id, name, days, seed, profile, origin,
    status: "running" | "done",
    lastUsed: iso,
    currentMealIdx, totalMeals,
    branchFrom: string | null,
  }
  ```
- status 派生: history.length === total_meals → "done", 否则 "running"
- lastUsed 取 state.started_at_real (或 state.last_advance_at_real 如果有)

#### POST /api/sandbox/sessions
- Request body: `{name: str, days: int = 7, seed: int, profile_label: str = "profile@v2", origin: "real_snapshot"|"blank", copy_real_data: bool = False}`
- 调 `sandbox.create_session(name=..., days=..., ..., root=ROOT, copy_real_data=copy_real_data)`
- copy_real_data=True 时复用 web_api._copy_real_data_to_sandbox 现有逻辑 (但落到新 session 目录,不是老 logs/sandbox/ 顶层)
- 返新建的 SessionMeta

#### GET /api/sandbox/sessions/{sid}
- 返 `FullSnapshot`:
  ```ts
  {
    meta: SessionMeta,
    clock: {idx, day, slot, total},
    history: Meal[],
    currentRecs: Rec[],            // 从 sessions/{sid}/last_recs.json 读;不存在则 []
    lastDecision: Decision | null, // 从 decisions/{current_meal_idx - 1}.json 读
    activeRules: {
      refine: [],                  // S-06a 阶段无 refine 持久态,返 []
      blacklist: [],               // 从 feedback_store.stopped 派生
    },
    taste: TasteEntry[],           // 从 long_term_prefs.json 派生
    keywords: KeywordEntry[],       // 同上
    recent: string[],              // 从 meal_log.jsonl 最近 4 顿
    fatigue: FatigueEntry[],       // 从 meal_log 同 restaurant_id 计数
  }
  ```
- 404 if sid not found
- 用 `with set_sandbox_session(sid):` 包装内部读取调用,确保 data_root 走对路径

#### 不在本任务范围 (S-06c 才有)
- POST /eat /skip /swap /refine
- GET /jobs/{job_id}

### Pydantic models (chisha/web_api.py)

新增 `SandboxCreateReq`, `SandboxSessionMeta`, `SandboxFullSnapshot`, `SandboxMeal`, `SandboxRec`, `SandboxDecision`, `SandboxActiveRules`, `SandboxTasteEntry`, ... — 直接对应 handoff §数据契约。

可以重用 chisha/schemas.py 已有类型,但 sandbox-lab 特有的 (Meal/Rec 简化版) 在 web_api 内新定义。

### tests

`tests/test_sandbox_sessions_api.py`:
- GET 空: list_sessions 返 [_default] (S-05 自动建)
- POST create "test1" → GET returns 2 conds
- GET /{sid} 不存在 → 404
- GET /{sid} 存在 → FullSnapshot shape 校验 (Pydantic validate)
- localhost-only: 假 IP header → 403
