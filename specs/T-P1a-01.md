# T-P1a-01 · methodology 硬契约下沉 L1（含 L0 三分接入）

参考: `docs/design_briefs/2026-05-18-refine-v2-faithful-refine.md` §3、§7。

## What
1. 把 `harvard_plate.yaml`（或 methodology 配置）里的健康硬契约（蔬菜占比 / 油上限 / 价格带）从 L2 weight 路径改为 L1 `hard_filter` 路径。
2. 在 `chisha/recall.py` 引入 L0 三分判定：
   - L0-A 医学风险类（过敏/药物冲突/孕期忌口） → 永不可破
   - L0-B 身份伦理类（清真/素食/宗教） → 永不可破
   - L0-C 普通健康类（油/糖/蔬菜占比/价格/减脂目标） → refine 明确表达可破
3. 加 profile schema 扩展支持 A/B 类标记（即使现在 profile 里没数据，schema 留位置）。
4. 与 T-00 落地的 `hard_filter_event` 接入：每次 L1 过滤都写 trace 事件。

## Why
- 修最大产品 bug：「原则派点餐执行外包」承诺没兑现 — 长期方法论（蔬菜≥50%/油上限）原本只是 L2 一个权重，与 popularity 同台竞争。
- 把硬契约下沉 L1，L2 只剩软偏好，设计层次清晰。
- L0 三分定义后续 refine 解除政策的判定。

## Done When
- L1 出去的 combo 100% 满足 L0 A/B 类约束（手工注入"花生过敏 + refine 想吃花生酱面"测例，系统拒绝并记 trace 事件）。
- L0 C 类约束：refine 文本明确表达时（如"今晚就放纵"）能解除（trace 里 `refine_override=true`）；不明确时仍生效。
- 空输入路径 baseline_l2_snapshot 严格 0 diff（无 refine 时旧逻辑路径不变 — 这是核心回归底线）。
- 新增至少 10 个 e2e 用例覆盖：L0-A 触发、L0-B 触发、L0-C 触发未解除、L0-C 被 refine 解除、methodology 各硬契约触发。
- `uv run pytest tests/ -q` 全绿。

## Affected files (预估)
- `chisha/recall.py`（核心改造）
- `chisha/methodology.py` 或 `harvard_plate.yaml`（硬契约配置）
- `chisha/score.py`（移除已下沉的权重）
- `profile.yaml` schema 扩展
- `chisha/trace_store.py`（写事件 — 用 T-00 的接口）
- 测试: `tests/test_recall_hard_filter.py`（新增）

## 红线
- 空输入路径（无 refine）必须 baseline_l2_snapshot 0 diff。这是 CONTRACTS.md 里的硬约束，违反直接 BLOCKED。
- 不引入新的 LLM 调用（refine 解除判定走文本规则即可，T-P1a-03 才上 LLM 多 slot 解析）。

## 不做
- 不实施破戒模式独立 feature（已被 L0 三分吸收，brief §3 取消）。
- 不动 L3 rerank prompt（那是 T-P1b-02 的事）。
