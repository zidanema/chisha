# T-P1b-02 · L3 prompt narrative 字段

参考: `docs/design_briefs/2026-05-18-refine-v2-faithful-refine.md` §8。**有 4 处联动，不是无侵入**。

## What
1. 在 `chisha/rerank.py` L3 prompt 加 `narrative` 输出字段。"为什么推这 5 道" ≤ 50 字。
   - 例: "今天阴雨 + 你近 2 餐高油 → 给你 X / Y / Z"
2. L3 输出 schema 加 narrative 字段（带 schema_version bump）。
3. 前端卡片展示 narrative（5 张卡上方）+ 每张卡 reason chip。
4. trace schema bump：保留旧 L3 trace 兼容（reading old traces without narrative 不崩）。
5. 命中 refine 时高亮 reason chip。

## Why
- "为什么推这 5 道" narrative 是用户信任的关键载体。
- 但 Codex v1 review 强调：narrative 在错误链路上是"欺骗放大器" — 必须在 P1a-01（硬契约下沉）+ P1a-02（召回参数对）之后才能上线，否则给不靠谱的系统装无敌嘴皮。

## Done When
- L3 输出 schema 加 narrative 字段（带 schema version bump）。
- 前端 5 张卡上方展示 narrative ≤ 50 字。
- 旧 trace 不带 narrative 仍能被 debug-ui Replay 模式回放（不崩）。
- 抽查 20 次生产推荐 narrative，全部能在 trace 里找到对应执行证据（不撒谎）。
- chrome-devtools-mcp 自驱验证 apps/web（强制）。
- `uv run pytest tests/ -q` + `cd apps/web && npm run typecheck` 全绿。
- 空输入路径 baseline_l2_snapshot 0 diff（trace schema 加字段但旧路径行为不变）。

## Affected files (预估)
- `chisha/rerank.py`（prompt + schema）
- `chisha/schemas.py` 或 `chisha/trace_store.py`（schema bump）
- `apps/web/src/components/RecommendationCard.tsx` 等
- `apps/web/src/components/NarrativeHeader.tsx`（新建）
- `apps/debug-ui/src/**`（如 V12 DAG 展示需要兼容）
- 测试: `tests/test_rerank_narrative.py`

## 红线
- prompt 必须显式标 cache_control（memory `chisha_llm_call_principles`）。
- 旧 trace 必须可回放（Replay 模式 D-079 不破）。
- narrative 文本必须有 trace 证据支撑 — 加单测验证 narrative 提到的"硬契约触发"在 trace `hard_filter_event` 里能找到。
- 改 apps/web/** 必须 chrome-devtools-mcp 自驱（CLAUDE.md 强制）。
- 旧 prompt 风格 mode（profile.yaml L3 model）必须保留切换能力。

## 不做
- 不做 reference 块的解析展示（T-P2-01）。
- 不做簇式输出（T-P2-02）。
- 不引入 tool_use 模式（保留现 text+JSON）。
