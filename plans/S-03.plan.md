## §0 v2 修订 (iter 1 fallback Opus BLOCKED 响应)

iter 1 Opus 标 5 条 BLOCKER. 响应:

### 修订 A — Per-session state map (BLOCKER #3)

iter 1 cited: plan §3.1 单一 `history` state vs §4.3 "选 SESSION_INITIAL_STATE_MAP 还是共用" 自相矛盾. branch action 必须 per-session 否则切回老 session 历史丢.

**v2 决策**: useSandbox 内部 `sessionStates: Map<string, SessionInternalState>` Map, 其中 `SessionInternalState = {history: Meal[], currentMealIdx: number, currentRecs: Rec[], lastDecision: Decision | null, dismissedDerivedBanners: Set<string>}`. Hook 对外暴露的 `history/currentRecs/currentMealIdx/lastDecision` 都从 `sessionStates.get(activeSessionId)` 取. 切 session 时**不**reset 这些字段, 而是切 map key.

初始化: `useState(() => initialSessionStatesFromMocks())`. 每个 SESSION mock 一份初始:
- s_new_l3 (running, days=7): history = mock HISTORY (前 4 顿), currentMealIdx=4, currentRecs=CURRENT_RECS, lastDecision=LAST_DECISION
- s_no_spicy (done, days=7) / s_cold (done, days=14): history = `expandHistoryForDoneSession(mockTotalMeals)` (mock helper, 复用 HISTORY[0..3] 模板循环填到 totalMeals 长度, 每条 idx/day/slot 正确), currentMealIdx=totalMeals, currentRecs=[], lastDecision=最后一顿派生.

`SessionInternalState.dismissedDerivedBanners` 字段去掉 (v2 修订 B: 派生 banner 不可 dismiss). 改后 SessionInternalState = `{history, currentMealIdx, currentRecs, lastDecision}` 4 字段.

### 修订 B — Banner dismiss per-meal scope, 不是 global (BLOCKER #1)

iter 1 cited: `dismissedBannerIds: Set` 用 stable id "refine_conflict" → eat 后新 recs 仍 conflict, 但 dismiss 集已含此 id → 用户永远看不到下一顿的 conflict 警告. 同理 review_mode banner.

**v2 决策**: 
- **派生 banner 不进 `dismissedBannerIds`**. 派生 banner 永远跟着 state (currentRecs.some(conflict) 为真就显示, 用户没法 dismiss)
- 派生 banner `dismissable: false` (UI 不渲 X 按钮 → 用户无 dismiss 入口) — 修 Banners 组件呈现逻辑? **No**: 不改组件; 派生 banner level/title 文案已经够 obvious, 用户不需要 dismiss
- transient banner (swap/refine_applied) **仍可** dismiss, id 用 `swap_${Date.now()}` / `refine_${Date.now()}` 唯一不重复. eat/skip 推进时清空整个 transientBanners list (id 自然失效, 不堆积)
- `dismissedBannerIds` state 移除 (无需此机制; transient list 清空即可, 派生不可 dismiss)

UI 影响: Banners 组件 dismissable=false 时按钮已不渲 (S-02 Banners.tsx 实现已查 dismissable prop). 派生 banner 不显 X.

### 修订 C — handleBranch / handleRollback boundary + 切 session 时 currentMealIdx (BLOCKER #2)

iter 1 cited: ConfirmModal 文案 "撤销 D{meal.day} {meal.slot} 及之后的 N 顿,该格将变为 current" → rollback 把 meal.idx 那顿也擦掉; branch 起点的 currentMealIdx 未定义. plan §7 risk 表漏了 "切 session 时重置 currentMealIdx".

**v2 决策**:
- **rollback(targetIdx)**: 在 active session state 内 `history = history.filter(h => h.idx < targetIdx)`; `currentMealIdx = targetIdx`; `currentRecs = freshRecsForMeal(targetIdx, sid)`; `lastDecision = null`. (与 ConfirmModal "该格变为 current" 一致)
- **branch(fromIdx, name)**: 拷一份当前 session state, 在新 sid 的 SessionInternalState 内 `history = oldHistory.filter(h => h.idx < fromIdx)`; `currentMealIdx = fromIdx`; `currentRecs = freshRecsForMeal(fromIdx, newSid)`; `lastDecision = null`. **老 session state 不动**. setActiveSessionId(newSid).
- 修订 A 后 "切 session 时 currentMealIdx 不 reset" 这个 bug 不存在: clock 用 `sessionStates.get(activeSessionId).currentMealIdx`, 自然跟着 active session 走.

### 修订 D — Done state UI (BLOCKER #4)

iter 1 cited: spec `Done When` 明确 "跑完整 14 顿". plan §3.2 "若已 >= totalMeals 不做" silent no-op + 没 done banner = dead button + 验收 #2 fail.

**v2 决策**:
- 加派生 banner `done`: `currentMealIdx >= totalMeals && !isInReviewMode` → `{id: "done_${activeSessionId}", level: "info", title: "Session 完成 ✓", detail: \`${totalMeals}/${totalMeals} 顿全部完成, 可分支或回滚\`, dismissable: false}`
- handleEat / handleSkip 在 done 状态时**仍 return early** (state 不变), 但 UI 已显 done banner 提示用户. RecCard "就这个 →" 按钮在 done 状态可禁用 — 但本任务不改 RecCard 组件 (会引入 prop 变更, violation §2 "不改 components"). 替代: DecisionArea 喂 `recs=[]` 当 done → DecisionArea 自然渲染 "无推荐" 状态. **Actually**: 不改 DecisionArea 喂逻辑, 喂 `currentRecs` 正常但 done banner 可见. 用户点 "就这个" 没反应 — 视觉差但功能上 done 信号在.
- 更好方案: App.tsx 渲染时 `done = currentMealIdx >= totalMeals; main = done ? <DoneView /> : <DecisionArea ... />`. 但加 DoneView 组件超出范围.
- 折中: App 主区 if done && !isInReviewMode → 显 done banner + 一个简单的 placeholder div ("跑完了 — 回到 timeline 选过去格查看 / 分支重跑"); 不渲 DecisionArea. ~5 行 inline JSX. 不算新组件.

### 修订 E — ConfirmModal stale state + 双 close (BLOCKER #5)

iter 1 cited: confirmMeal 作独立 state 会 stale; handleRollback 后 ConfirmModal 自身又 onClose → 顺序不明.

**v2 决策**:
- 不存 `confirmMeal` 独立 state. 改派生: `confirmMeal = selectedCellIdx !== null ? history[selectedCellIdx] : null` (从 active session history 取)
- handleRollback / handleBranch 内**不**调 closeConfirm (ConfirmModal 自身 `onConfirm?.(...); onClose?.()` 内部已关; handleRollback 关心 state, ConfirmModal 关心自己关). plan §3.2 step 6 "confirmKind = null (关 modal)" 删
- 顺序: ConfirmModal onConfirm → `sb.handleRollback(idx)` (state 改) → ConfirmModal `onClose()` → `sb.closeConfirm()` 设 confirmKind=null. 一致 deterministic.
- handleRollback step 4 `selectedCellIdx = null` 保留 (rollback 后 selectedCell 已超出新 history 范围, 必须清)

### 修订 F — demo-* query 兼容保留 (NB-1 升级)

## §3 Hook contract

### 3.1 `useSandbox()` 返回 store + actions

```typescript
interface UseSandboxResult {
  // ── store
  sessions: SessionMeta[];
  activeSessionId: string;
  clock: Clock;                          // 派生自 currentMealIdx
  currentMealIdx: number;                // 0..total-1; 推进时持久 (S-03 内 useState)
  totalMeals: number;                    // 等于 activeSession.days * 2
  history: Meal[];                       // sub-array of 完整 session history
  currentRecs: Rec[];                    // session 的当前 5 mocks
  lastDecision: Decision | null;
  // 回顾模式
  selectedCellIdx: number | null;
  // banners
  transientBanners: BannerEntry[];       // swap_applied / refine_applied 等
  derivedBanners: BannerEntry[];         // refine_conflict / review_mode / done (派生)
  banners: BannerEntry[];                // = derived + transient (v2 修订 B: 无 dismiss filter)
  // modals
  summaryOpen: boolean;
  confirmKind: ConfirmKind | null;
  confirmMeal: Meal | null;              // rollback 用 history[selectedCellIdx]; branch 同
  refineModalOpen: boolean;

  // ── actions
  setActiveSessionId(id: string): void;
  handleEat(rec: Rec): void;
  handleSkip(): void;
  handleSwap(): void;
  handleRefine(text: string): void;
  selectCell(idx: number | null): void;
  handleRollback(idx: number): void;
  handleBranch(fromIdx: number, name: string): void;
  dismissBanner(id: string): void;
  openSummary(): void;
  closeSummary(): void;
  openConfirm(kind: ConfirmKind): void;
  closeConfirm(): void;
  openRefineModal(): void;
  closeRefineModal(): void;
  // 派生 (review mode 主区切换)
  isInReviewMode: boolean;               // selectedCellIdx !== null && !== clock.idx
  reviewMeal: Meal | null;               // history[selectedCellIdx] when in review mode
}
```

### 3.2 行为细节 (spec + design brief)

**handleEat(rec)**:
1. 加 history entry: `{idx: clock.idx, day: clock.day, slot: clock.slot, state: "eat", dish: rec.name, score: rec.l3, flags: rec.conflict ? ["conflict"] : [], altScores: derive from currentRecs}`
2. `currentMealIdx += 1` (若已 >= totalMeals 不做)
3. `lastDecision = buildDecisionMock(rec, oldClock)` (S-06b style)
4. `currentRecs = freshRecsForMeal(newIdx, activeSessionId)` (新一组)
5. `transientBanners = []` (清 swap/refine applied)
6. `selectedCellIdx = null` (eat 必须在当前 cell)

**handleSkip()**: 类似 handleEat 但 state="skip"/dish="—"/score=null; lastDecision = `{when, pick:"(跳过)", rank:"—", l3:"—", diff:[], implications:[{field:"—", text:"跳过未触发学习,下一顿系统状态不变"}]}`

**handleSwap()**:
1. `currentRecs = freshRecsForMeal(clock.idx, activeSessionId)` 重排 (mock helper 内 random shuffle / 取下一组)
2. 加 transient banner: `{id: "swap_" + Date.now(), level: "info", title: "已换一组", detail: "新拉了 5 条 (mock)", dismissable: true}`
3. **不**推进 idx

**handleRefine(text)**:
1. `currentRecs = freshRecsForMeal(clock.idx, activeSessionId).map(r => ({...r, intent: text.slice(0,8)}))` (mock 把 intent 写成用户文本)
2. transient banner: `{id: "refine_" + Date.now(), level: "info", title: "Refine 已应用", detail: \`本轮内 \"${text}\"; 下顿自动清\`, dismissable: true}`
3. 不推进 idx

**selectCell(idx)**:
- idx == null → 退出 review, `selectedCellIdx = null`
- idx == clock.idx → 不进 review, `selectedCellIdx = null` (按当前 cell 不算回顾)
- 否则 `selectedCellIdx = idx`

**clearSelection()**: `selectCell(null)`

**handleRollback(idx)**:
1. `history = history.filter(h => h.idx < idx)` (裁掉 idx 及之后)
2. `currentMealIdx = idx`
3. `currentRecs = freshRecsForMeal(idx, activeSessionId)`
4. `selectedCellIdx = null`
5. `lastDecision = null` (rollback 后 D panel 暂无新 decision)
6. `confirmKind = null` (关 modal)
7. transient banner: `{id: "rollback_" + Date.now(), level: "warn", title: "已回滚", detail: \`session 已截到 D{day}{slot} 之前\`, dismissable: true}`

**handleBranch(fromIdx, name)**:
1. 新 sid = `sessions[活动].id + "_branch_" + Date.now()` (短 hex 后缀)
2. 新 SessionMeta: `{id: newSid, name: \`branch · ${name}\`, days, seed, profile, origin: "branch", status: "running", lastUsed: "刚刚"}`
3. push 到 sessions, setActiveSessionId(newSid)
4. clone history to new session map, 裁到 fromIdx (类似 rollback)
5. confirmKind=null, banner "已分支"

**dismissBanner(id)**:
- 加到 `dismissedBannerIds` Set; banners 渲染时过滤掉
- transient: 同时从 transientBanners 移除 (清 state)

### 3.3 派生 banners 规则

`derivedBanners` (v2 修订 B: 全部 `dismissable: false`, 不进 dismiss filter):
- 当 `!isInReviewMode && currentRecs.some(r => r.conflict)` → 加一条 `{id: "refine_conflict", level: "warn", title: "Refine 冲突", detail: \`推荐 #\${rank} \${name} 违反 active refine '\${conflictRule}'\`, dismissable: false}`
- review 模式 → `{id: "review_mode", level: "review", title: "回顾模式 · D{day}{slot}", detail: "Esc 退出 · 此为只读快照", dismissable: false}`
- done 状态 (v2 修订 D) → `{id: "done_${activeSessionId}", level: "info", title: "Session 完成 ✓", detail: \`${totalMeals}/${totalMeals} 顿全部完成, 可分支或回滚\`, dismissable: false}`

`banners = [...derivedBanners, ...transientBanners]` (无 dismiss filter; transient 由 advance / 同类 push 自然清, derived 永显)

### 3.4 `useKeyboard({onEsc})`

```typescript
export function useKeyboard(opts: { onEsc: () => void }): void {
  // v2 修订 H: ref-stable handler 避免 inline arrow 反复 attach/detach
  const handlerRef = useRef(opts.onEsc);
  handlerRef.current = opts.onEsc;
  useEffect(() => {
    function h(e: KeyboardEvent) {
      if (e.key === "Escape") handlerRef.current();
    }
    window.addEventListener("keydown", h);
    return () => window.removeEventListener("keydown", h);
  }, []);   // 仅 mount/unmount, deps 空
}
```

## §4 Mock helpers (sbxMocks.ts append)

### 4.1 `freshRecsForMeal(idx: number, sessionId: string): Rec[]`

简化策略: 用 `idx + hashSession(sessionId)` 作随机种子, shuffle CURRENT_RECS 顺序 + 调整 rank 字段; 每次返不同顺序但同 5 个 mock 数据. 这能让 eat 后用户看到 "推荐变了" 视觉反馈, 同时 deterministic 跑 chrome-devtools-mcp.

```typescript
function hashStr(s: string): number {
  let h = 0;
  for (const c of s) h = (h * 31 + c.charCodeAt(0)) | 0;
  return Math.abs(h);
}

export function freshRecsForMeal(idx: number, sessionId: string): Rec[] {
  const seed = (idx * 1009 + hashStr(sessionId)) % 5;
  const rotated = [...CURRENT_RECS.slice(seed), ...CURRENT_RECS.slice(0, seed)];
  return rotated.map((r, i) => ({ ...r, rank: i + 1 }));
}
```

### 4.3 Per-session initial state map

不实现复杂 per-session 历史 (会让 mock 变臃肿). S-03 内**所有 session 共用** `HISTORY` 初始 + `CURRENT_RECS`. 切 session 时:
- session.status == "done" → history 全 14 顿 (从 HISTORY 扩展, mock 补 10 顿 dummy)
- session.status == "running" → history 切前 4 顿 (现有 HISTORY)

简化版: 切 session 时直接用 mock `HISTORY` (前 4) + currentMealIdx=4, 不针对每个 session 做不同初始数据. 切换重渲染但内容相同 → 满足 spec "整页重渲染对应 mock" (即使 mock 同).

更细致的做法 (推荐): `SESSION_INITIAL_STATE_MAP` 字典, 每个 sessionId 一份初始 history+idx. 实现起来 ~30 行, 不写复杂逻辑.
