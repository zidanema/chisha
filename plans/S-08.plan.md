## Goal
把 S-03 mock-only 的 sandbox-lab 前端切到真后端 (S-07 完整端点). 11 条验收清单在 backend online (mock_recommend=1) + backend offline 两个模式下全过. typecheck + pytest 全绿.

2. **新建** `apps/sandbox-lab/src/api/backend-types.ts` — backend dict shape

## Backend dict shape (实测自 S-07 web_api.py)

### SandboxSessionMeta (GET /sandbox/sessions list entry)
`{sid, is_default, created_at, size_bytes, has_state}` — **list view 元数据**.

### SessionMetaFull (FullSnapshotResp.meta)
`{sid, name, days, seed, profile, origin, status, lastUsed, currentMealIdx, totalMeals, branchFrom}` — 与前端 SessionMeta 几乎对齐 (`id` vs `sid`, `lastUsed` 字段名一致).

### SandboxClock
`{idx, day, slot: "lunch"|"dinner", total}` — **slot 中英文要翻**.

### History entry (backend)
`{idx, state, dish, session_id, accepted_at, rank, reason?}` — 缺 `day/slot/score/flags/altScores`. adapter 派生.

### currentRecs item (backend format_v2_to_rec)
`{rank, name, venue, dishes, price, l2, l3, boost, intent, l1Hits, explore, conflict, meta:{eta,dist,protein,oil}, why, id}` — 与前端 Rec **完全对齐** (含 id 额外字段, 兼容).

### lastDecision (backend build_decision)
`{when, pick, rank: int|"—", l3: int|"—", diff, implications}` — 与前端 Decision **完全对齐**.

### activeRules (BACKEND only — 本任务不接入前端视图)
backend FullSnapshot 返 `{refine, blacklist}`. **S-08 不接 BPanel** (Iter 1 issue 2 决议): RightCol 维持 MOCK 不入 scope. adapter 不导出 activeRules, useSandbox 也不 expose. future S-11 再连.

### Eat resp / Skip resp / Refine resp / Swap resp
- eat: `{job_id, status:"running", new_meal_idx, meal_idx_eaten}` → 启 polling
- skip: `{new_meal_idx, meal_idx_skipped, decision}` → 同步, decision 直接可用
- swap: `{currentRecs, recommend_session_id}` → 同步
- refine: `{currentRecs, recommend_session_id, activeRules}` → 同步, **只取 currentRecs**, activeRules 丢弃
- jobs/{jid}: `{status: "pending"|"running"|"done"|"failed"|"cancelled_by_rollback", started_at, ended_at?, result?:{decision, meal_idx}, error?}`. **`cancelled_by_rollback` 当 terminal failure 处理** (toast 提示 + 不推进 state + clearInterval)

## Step-by-step

### Step 1 · backend-types.ts
照搬 debug-ui 风格: 单文件列 5 个 backend type alias (BackendSessionsListItem / BackendFullSnapshot / BackendEatResp / BackendJobInfo / 其他). 不导入 sandbox 视图 type, 避免循环.

### Step 2 · client.ts
- `fetchJson<T>(path, init)` 仿 debug-ui (含 ApiError + 0 status NETWORK case)
- `pingBackend(): Promise<boolean>` — GET /api/sandbox/sessions, 返 ok
- 11 个函数: `listSessions`, `getFullSnapshot(sid)`, `postRecs(sid, {meal_type?, mock_recommend?})`, `postEat(sid, rec_rank)`, `postSkip(sid, reason?)`, `postSwap(sid, exclude_ids[])`, `postRefine(sid, text)`, `getJob(sid, jid)`, `postRollback(sid, meal_idx)`, `postBranch(sid, from_meal_idx, name)`, `createSession(sid, days)` (后两个本任务可能用不到 createSession, 但留入口)
- **mock_recommend query 规则**: 读 `window.location.search`, 若有 `?real=1` → 不加 `mock_recommend`, 走真后端 LLM; 否则默认带 `?mock_recommend=1` (sandbox-lab 验收清单约定). 仅作用于 /recs /swap /refine 三个端点 (其他端点 backend 没 mock_recommend 参数). 注意: backend `last_recs.json` 携带 `is_mock` flag, 若 mount 时仍残留 mock 数据, 切 `?real=1` 后要么手动 /recs 重置, 要么默认 mock 路径 — plan 不自动清理, 假定志丹切模式时自己 /recs.

### Step 3 · adapter.ts
导出 5 函数:
- `backendSessionsToMeta(list, snapshotsBySid?)` → 前端 SessionMeta[]. list view 只给 sid/created_at/size_bytes, 需要走 snapshot 才能拿 days/seed/name. 实现: list 各项调 getFullSnapshot 拼成 SessionMeta. **代价**: N 次 fetch — 接受 (sandbox 桶通常 < 5 个, 一次性).
  - Edge: snapshot 404 (sid has no state.json) → skip / 给 default 占位 `{name:"(空桶)", days:7, origin:"空白", ...}`.
- `backendFullSnapshotToStore(snap)` → `{meta: SessionMeta, clock: Clock, history: Meal[], currentRecs: Rec[], lastDecision: Decision|null}`. 子转换 (**不输出 activeRules** — 见上 §activeRules):
  - clock: `slot: lunch→"午"|dinner→"晚"`
  - history: `idx → {idx, day:Math.floor(idx/2)+1, slot:idx%2===0?"午":"晚", state, dish:dish??"—", score:null, flags:[], altScores:[]}`. state="skip" 时 dish 用 "—".
  - currentRecs: 直透 (字段完全对齐), 仅 round l1Hits/dishes 兜底 []
  - lastDecision: 直透 (字段对齐)
  - meta.origin: `backend "blank" → "空白"`, `"real_snapshot"/真值 → "真实快照"`, 默认 fallback `"空白"`
- `backendRecToFrontend(c)` — 单独一个供 swap/refine resp 用 (currentRecs 字段)
- `frontendDecisionFromBackend(d)` — 单独 (eat polling 完成时 _JOB_TABLE.result.decision 转)
- `backendJobToStatus(j)` → `"running"|"done"|"failed"|"cancelled"`. 把 backend 的 `"cancelled_by_rollback"` 映射到 `"cancelled"`, `"pending"` 也归 `"running"`.

### Step 4 · useSandbox.ts
保留现有 mock 路径 (backendOnline=false fallback 完全保持 S-03 行为). 新增:
- state: `backendOnline: boolean | null` (null = pinging), `polling: {action, jobId, sid} | null`, `error: string | null`
- **interval lifetime**: 用 `useEffect([polling])` 管理 setInterval. 进 effect 启 timer, cleanup 调 clearInterval. polling=null 时不启. **切 session / rollback / branch / unmount 时**通过 setPolling(null) 触发 cleanup. polling tick 内拿 latest sid: 若 `polling.sid !== activeSessionId` (用户切走) → setPolling(null) 直接弃, 不写 setStore.
- effect on mount: `pingBackend()` → setBackendOnline. 成功 → `listSessions()`. **default-bucket 规避** (Iter 3 issue 3): backend rollback/branch 拒 `_default` (web_api.py:2221/2256). 选 active sid 时:
  - 若 list 含非 default sid (`is_default=false`) → 选第一个 (or 上次 localStorage 记的)
  - 若 list 只有 `_default` → 自动 `createSession("s08_e2e", days=7)` (409 已存在视为成功) → 拉新 sid 的 FullSnapshot
  - 不允许 active=`_default` 进入正常 e2e 流程
  - 之后 `listSessions()` + 并发 `getFullSnapshot(sid)` → adapter 填整套 sessionStates map + sessions list (覆盖 SESSIONS mock). 对 active sid 调 `ensureRecs`.
- handleEat/skip/swap/refine/rollback/branch 改造:
  - **`ensureRecs(sid, snap)` helper**: 若 `snap.currentRecs.length === 0 && !isDone(snap)` → `POST /recs?mock_recommend=1` (走 client withMock) → 用 resp.currentRecs 覆盖到 store. 否则直透.
  - eat: POST /eat → 收 job_id → setPolling({action:"eat", jobId, sid: activeSessionId}) → useEffect interval 1500ms → `getJob` → status `done` → `getFullSnapshot(sid)` → `ensureRecs(sid, snap)` → setStore + setPolling(null). status `failed`/`cancelled` → setError + setPolling(null) + 不推进 state.
  - skip: POST /skip 同步 → resp 含 decision → `getFullSnapshot(sid)` → ensureRecs → setStore.
  - swap: POST /swap 同步 → resp 含 currentRecs → updateSessionState.currentRecs (直接覆盖, 不动 history/decision).
  - refine: POST /refine 同步 → resp 含 currentRecs → updateSessionState.currentRecs + transient banner. **丢弃 resp.activeRules** (RightCol 不消费).
  - rollback: 入口先 `setPolling(null)` (Iter 3 race fix: 避免 backend op-lock 409). 然后 POST /rollback (resp = FullSnapshot) → ensureRecs → setStore.
  - branch: 入口先 `setPolling(null)`. 然后 POST /branch (resp = new SandboxSessionMeta) → `getFullSnapshot(new_sid)` → ensureRecs → push session list + setActive.
- offline: 现有 mock 不动.
- `?real=1` query: 见 Step 2 — client 透传, useSandbox 不需改.

### Step 5 · App.tsx

- polling 期间: `sb.polling ? <"处理中..." banner + 5 个 skeleton 卡 (在 DecisionArea 位置)>` — 不改 DecisionArea 组件, App.tsx 顶层条件渲染 5 个 div.skeleton 元素覆盖.

## Regression risk · medium
- 不动后端 → backend 测试 0 风险
- 前端只动 useSandbox / App / 新增 api/. mock 路径必须保留 fallback (offline 兜底).
- Polling 内存泄漏: 必须 useEffect cleanup clearInterval; 切 session 时清当前 polling.
- Race: 切 session 中 polling 未完成 → polling 完成后用 sid 守门, 不当前 session 直接丢结果.
- adapter slot 翻译错 → timeline 显示错日序. 单元: 写 console 一次性 sanity log (临时 dev), 跑通后删.
