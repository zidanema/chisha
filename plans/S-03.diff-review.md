### Sketchy Pattern Audit

**`__DEFAULT_TOTAL_MEALS__` export (useSandbox.ts:495)**
- Dead code — not imported anywhere in App.tsx or components. Export is harmless (tree-shaken in build, `npm run build` confirmed 184KB bundle). Comment says "helps any future caller." Low risk. Non-blocking.

**IDLE_DECISION fallback (App.tsx:21-28)**
- `RightCol` prop `decision: Decision` is non-nullable in the component signature. `sb.lastDecision ?? IDLE_DECISION` is the correct pattern given plan constraint "不动 components." Correct trade-off.

**Banners onDismiss=sb.dismissTransient**
- `Banners` component calls `onDismiss(b.id)` only when `b.dismissable` is true. Derived banners have `dismissable: false`, so `dismissTransient` is never called for them. Transient banners (swap/refine/rollback/branch) all use `dismissable: true`. Correct wiring.

**TopBar onNewSession → branch from currentMealIdx**
- `App.tsx:115`: `() => sb.handleBranch(sb.currentMealIdx, "新-${Date.now() % 10000}")`. This calls branch from the **current** (not yet eaten) idx. Plan Q3 accepted this as the lightweight equivalent. Branch will create session with `history = history.filter(h => h.idx < currentMealIdx)` — same as current history. New session has same state as active session at branch point. Acceptable mock behavior.

**s_cold days=14 → totalMeals=28, initialIdx=14 — not truly done**
- `sbxMocks.ts:45-48`: `s_cold: {days: 14, status: "done"}`. `totalMeals = 14 * 2 = 28`. `initialSessionStates()` sets `currentMealIdx = total = 28`, `history = expandHistoryForDoneSession(28)` (28 entries). `isDone = 28 >= 28 = true`. **Actually correct**: the code uses `sess.days * 2` as `total` and passes it to `expandHistoryForDoneSession`, then sets `currentMealIdx = total`. So s_cold correctly shows as done with 28/28. The iter-2 NB-3 concern about idx=14 was a plan-level concern that the implementation resolved correctly by using `const total = sess.days * 2` uniformly.
