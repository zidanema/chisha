### 修订 A — `advance_meal` 用**自有 sid-aware path helper**, **不依赖** `state()` / `_state_path()`

iter 1 cited: `state()` line 100 `del session_id`是 S-04 stub, ContextVar 不被消费; `_state_path()` 无 sid 参数. 所以 plan v1 §3.2 step 2 "用 `set_sandbox_session(sid)` 包" 是空操作, 非 default sid 仍写 default 桶 state.json.

**v2 决策**: `advance_meal` **直接派生 sid-aware state path**, 不调 `state()` / `_state_path()`. 新加内部 helper `_state_path_for_sid(sid, root)`:
- `sid is None or sid == _DEFAULT_SID` → `_state_path(root)` (即 `logs/sandbox/state.json`, default 扁平)
- 非 default sid → `_sessions_root(root) / sid / "state.json"` (即 `logs/sandbox/sessions/{sid}/state.json`)

`advance_meal` 内部:
```python
def advance_meal(*, sid: str | None = None, root: Path | None = None) -> dict:
    state_p = _state_path_for_sid(sid, root)
    with _STATE_LOCK:
        if not state_p.exists():
            raise RuntimeError(f"sandbox state not found at {state_p}; call init() first")
        try:
            s = json.loads(state_p.read_text(encoding="utf-8"))
        except (OSError, json.JSONDecodeError) as e:
            raise RuntimeError(f"sandbox state corrupt at {state_p}: {e}")
        if not isinstance(s, dict) or not s.get("enabled"):
            raise RuntimeError("sandbox not enabled; call init() first")
        cur_idx = int(s.get("current_meal_idx", 0))
        total = int(s.get("total_meals", 14))
        new_idx = cur_idx + 1
        if new_idx > total:
            raise RuntimeError(f"meal_idx {new_idx} exceeds total {total}")
        s["current_meal_idx"] = new_idx
        if cur_idx % 2 == 1:  # was dinner
            cur_date = dt.date.fromisoformat(s["current_date"])
            s["current_date"] = (cur_date + dt.timedelta(days=1)).isoformat()
            s["day_index"] = int(s.get("day_index", 1)) + 1
        state_p.write_text(
            json.dumps(s, ensure_ascii=False, indent=2),
            encoding="utf-8",
        )
        return s
```

**trade-off**:
- (+) 完全 bypass S-04 stub (`state()` 仍 `del session_id`), 不破坏 S-04 契约
- (+) 路径派生显式, 测试可断言 `sessions/{sid}/state.json` 真实写入
- (+) 不需要改 `state()` / `_state_path()` 任何行为, 减少回归面 (S-06c/inspect 不受影响)
- (-) 多一份 path 派生逻辑 (与未来 `state()` sid-aware 改造可能重合) — 标记 `TODO(S-06c): state()/_state_path() sid-aware refactor` 让 S-06c 决定是否统一
- (-) 不复用 `_state_path` 的 "_default" 处理逻辑 — 但本任务 `_DEFAULT_SID = "_default"` 常量, 一处判断够清晰

### 修订 B — `total_meals` 边界条件: `> total` (与 spec 一致), 文档说明语义

iter 1 FINDING-3 指 `new_idx > total` 在 default 14 下允许 idx=14, `meal_idx_to_slot(14)=(lunch, 8)` 超 7 天.

**v2 决策**: 沿用 spec `if new_idx > total: raise`. **明确语义**: `total_meals=14` 表示"全部消费完后 current_meal_idx 终态=14, 表示 done 状态; idx ∈ [0, 14] 合法 (0-13 = 待消费餐位, 14 = 已 done)". `meal_idx_to_slot` **只对 [0, 13] 有效**, 调用方 (前端 / S-06c) 在 idx=14 时不该再问"下一顿是啥". 这与 spec line 41 一致.

**文档落地**: `advance_meal` docstring 注明 "total_meals 是 done sentinel, idx ∈ [0, total_meals], meal_idx_to_slot 调用方需自行处理 idx == total_meals 的终态". `meal_idx_to_slot` 不改, 不加 idx >= 14 校验 (会触发 ValueError, 但前端调用方约束).

### 修订 C — build_decision `history` 契约写入 docstring

iter 1 FINDING (build_decision history 假设含本顿). **v2 决策**: docstring 明确 "history 列表的 `[-1]` 元素是本顿 (即 picked_rec 对应的那一顿); same_count 计算 `history[:-1]` 内同 dish 出现次数". S-06c 端点调用时按此约定填充.

## §3 API contract

### 3.1 `meal_idx_to_slot(idx: int) -> tuple[Literal["lunch","dinner"], int]`
- idx >= 0, 否则 ValueError.
- 返 `(meal_type, day_1based)`:
  - idx % 2 == 0 → ("lunch", idx//2 + 1)
  - idx % 2 == 1 → ("dinner", idx//2 + 1)
- 例: 0→(lunch,1), 1→(dinner,1), 2→(lunch,2), 13→(dinner,7)

### 3.2 `advance_meal(*, sid: str | None = None, root: Path | None = None) -> dict`

- 行为 (v2 修订 A: 直接派生 sid-aware path, 不调 `state()`):
  1. `state_p = _state_path_for_sid(sid, root)` (新 helper, sid=None/"_default" → default 扁平 `logs/sandbox/state.json`, 非 default sid → `logs/sandbox/sessions/{sid}/state.json`).
  2. 取 `_STATE_LOCK`.
  3. `state_p.exists() == False` → RuntimeError.
  4. 读 JSON, 异常 → RuntimeError("sandbox state corrupt").
  5. `not s["enabled"]` → RuntimeError("sandbox not enabled").
  6. `cur_idx = int(s.get("current_meal_idx", 0))`, `total = int(s.get("total_meals", 14))`. `new_idx = cur_idx + 1`. 若 `new_idx > total` → RuntimeError (v2 修订 B: total 是 done sentinel, idx ∈ [0, total]).
  7. `s["current_meal_idx"] = new_idx`. 若 `cur_idx % 2 == 1` (was dinner) → date+=1, day_index+=1.
  8. 写回 state.json (atomic via 字符串 dump).
  9. 返新 state dict.

- sid 非空 + sandbox disabled at root → 仍 raise (state.enabled=False).
- 不调 `_reject_nondefault_sid`: 本函数就是 S-06b 引入的多 session 安全 advance, 非 default sid 合法.

### 3.2.a 新 helper `_state_path_for_sid(sid, root)` (v2 修订 A)

```python
def _state_path_for_sid(sid: str | None, root: Path | None = None) -> Path:
    """v2 修订 A: sid-aware state.json path. 不依赖 ContextVar.

    - sid None / "_default" → logs/sandbox/state.json (default 扁平, D-077)
    - 非 default sid → logs/sandbox/sessions/{sid}/state.json

    本 helper 仅供 advance_meal 用. 既有 state() / _state_path() 不动
    (S-04 stub 沿用, S-06c 决定是否统一 sid-aware 化).
    """
    from chisha.sandbox_context import _DEFAULT_SID
    if sid is None or sid == _DEFAULT_SID:
        return _state_path(root)
    return _sessions_root(root) / sid / "state.json"
```

### 3.3 `build_decision(*, sid, meal_idx, picked_rec, prev_long_term_prefs, new_long_term_prefs, history, root=None) -> dict`

- picked_rec=None → skip Decision (when/pick="(跳过)"/rank="—"/l3="—"/diff=[]/implications=[skip 文案]).
- picked_rec 非空:
  - `when = f"D{day} {午|晚}"` (从 meal_idx 派生).
  - `pick = f"{picked_rec['name']} · {' + '.join(picked_rec.get('dishes',[]))}"` (无 dishes → 只 name).
  - `rank` = picked_rec.get("rank"), `l3` = picked_rec.get("l3").
  - **diff 构造**:
    - 必 add: `recent_dishes` `+ [{name}]`
    - 必 add: `fatigue.{name}` from `str(same_count)` (>0) / "—" (=0); to `str(same_count + 1)`. `same_count = sum(1 for h in history[:-1] if h.get("dish") == name)` (history 含本顿, 所以 [:-1])
    - 选 add: 对比 prev_boost / new_boost 每个 token, `abs(new - old) > 1e-3` 时落 `taste.<{token}>` up/dn (`kind=up if v>old_v else dn`, delta 含符号 ±0.NN)
  - **implications 构造**:
    - 每个 taste up/dn → `下顿 L2 给同类 {delta} 倾向` (注意 delta 已含符号)
    - 每个 fatigue add → `下顿同菜折 0.95×`
    - 其它 add (如 recent_dishes) → 不生成 implication

- 纯函数, 不读盘, 不依赖 root (root 只是 future-proof; 本任务实现不用).
- **v2 修订 C**: history 契约: 列表 `[-1]` = 本顿 (即 picked_rec 对应那顿); `same_count = sum(1 for h in history[:-1] if h.get("dish") == picked_rec['name'])`. S-06c 端点按此填充.

## §5 Regression risks + rollback

| 风险 | 严重度 | 缓解 |
|---|---|---|
| advance_meal 与既有 advance() 行为重合 (cur_idx 推进 vs 整天跳跃) | 低 | 两个函数语义不同, advance 整天 (整 day_index += days), advance_meal 单顿. 命名清晰, doc 区分. S-06c eat/skip 端点用 advance_meal; reset 后再开新 sandbox 仍用 init() |
| state.json 多了 current_meal_idx / total_meals 字段, 影响 list_sessions / inspect | 低 | 都是 `.get(..., default)` 兼容; init 不写 → 老 state 不破坏 |
| build_decision diff 顺序敏感 | 低 | 测试断言含 specific items, 不锁定顺序 (set-like 比较) |

### Rollback
`git revert HEAD` 即恢复. 新模块 sandbox_decision_diff.py 删干净, sandbox.py 两个 helper 删. 无 schema 变更.
