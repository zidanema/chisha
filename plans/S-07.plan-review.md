## Iter 1

Blocking issues:

1. Rollback atomicity is explicitly weakened below the spec, `plans/S-07.plan.md:31`
   Spec requires all rollback steps to be atomic and says failures must either roll back already-mutated parts or use a single commit (`specs/S-07.md:13`, `specs/S-07.md:33`). The plan accepts `history.json` / `meal_to_trace.json` being truncated while `state.json` remains old (`plans/S-07.plan.md:34`), which creates exactly the split-brain state the spec forbids.

2. In-flight background jobs can rewrite truncated decisions or prefs after rollback, `plans/S-07.plan.md:45`
   `/eat` starts a background task after advancing state (`chisha/web_api.py:1454`) and that task later refreshes L1 prefs and writes `decisions/{meal_idx}.json` (`chisha/web_api.py:1329`, `chisha/web_api.py:1335`, `chisha/web_api.py:1352`). Clearing `_JOB_TABLE` after commit only removes bookkeeping; it does not stop or join the worker (`chisha/web_api.py:1101`). Contracts require sandbox lifecycle-style mutations to coordinate with L1 extraction to avoid stale writes (`docs/CONTRACTS.md:98`).

3. The test plan cannot verify trace deletion or meal_log truncation because it uses only mock recommendations, `plans/S-07.plan.md:98`
   Mock `/recs` only creates a synthetic `mock_*` id and candidate payload, not a trace file (`chisha/web_api.py:1263`). Mock `/eat` skips `_impl_accept`, so it does not write `meal_log.jsonl` (`chisha/web_api.py:1416`). Yet S-07 must delete trace files and trim meal_log (`specs/S-07.md:17`, `specs/S-07.md:20`), and the listed tests only assert `meal_to_trace` id replacement (`plans/S-07.plan.md:106`).

4. Rollback response shape assumes a FullSnapshot path/model that is not present in the current web API, `plans/S-07.plan.md:88`
   The design brief requires `GET /api/sandbox/sessions/{sid}` to return `FullSnapshot` (`docs/design_briefs/2026-05-19-sandbox-lab.md:120`), and S-07 spec says rollback returns a new FullSnapshot (`specs/S-07.md:31`). Current `web_api.py` only defines list/create/delete/rename session routes around this area (`chisha/web_api.py:1022`, `chisha/web_api.py:1030`, `chisha/web_api.py:1051`, `chisha/web_api.py:1063`) and no `FullSnapshot` model is defined; a "new state" dict is not the downstream contract.

5. Branching copies stale rollback temp state by design, `plans/S-07.plan.md:51`
   The plan says `copytree` includes `.rollback_tmp` and treats that as harmless, but rollback uses that directory as its transactional staging area (`plans/S-07.plan.md:21`). Copying a leftover staging directory into the new branch means branch rollback can inherit partial artifacts from a prior failed operation, conflicting with the design brief's requirement that branch copies a session and then explicitly trims future state (`docs/design_briefs/2026-05-19-sandbox-lab.md:127`).

Faithful Refine check: no direct refine implementation is in the affected-file list (`plans/S-07.plan.md:113`), and the existing refine endpoint remains in `chisha/web_api.py:1619`; the plan does not change refine ranking semantics. The blocking risks are sandbox state/trace/prefs consistency, not refine fidelity.

VERDICT: BLOCKED
