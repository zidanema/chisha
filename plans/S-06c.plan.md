### 修订 A — `sandbox.create_session` 必须 seed state.json (BLOCKER #1)

**问题**: S-06a `create_session` 只 mkdir 桶, 不写 state.json. `_state_path_for_sid("s1") → sessions/s1/state.json` 在 /recs /eat 调用时不存在 → `advance_meal` raise. 测试通过是因手工写 state.json (S-06b plan §4.2), production POST /api/sandbox/sessions 创建的桶都会崩.

**v2 决策**: **改 `chisha/sandbox.py:create_session`**, 在 mkdir 后 seed state.json. 加可选参数:

```python
def create_session(
    sid: str,
    *,
    root: Path | None = None,
    days: int = 7,
    start_date: dt.date | str | None = None,  # 默认 dt.date.today()
    seed_state: bool = True,                   # 兼容老 caller (S-06a 测试)
) -> dict:
    """S-06a 扩 (v2 修订): mkdir + 可选 seed state.json."""
    # ... 现有 _validate_sid + mkdir 不变 ...
    new_dir.mkdir(parents=True, exist_ok=False)
    if seed_state:
        if isinstance(start_date, str):
            start_date = dt.date.fromisoformat(start_date)
        start_date = start_date or dt.date.today()
        state_p = new_dir / "state.json"
        state_p.write_text(json.dumps({
            "enabled": True,
            "current_date": start_date.isoformat(),
            "day_index": 1,
            "current_meal_idx": 0,
            "total_meals": days * 2,
            "started_at_real": _now_real_iso(),
            "started_at_virtual": start_date.isoformat(),
            "sid": sid,
        }, ensure_ascii=False, indent=2), encoding="utf-8")
    return {"sid": sid, ..., "has_state": seed_state}  # has_state 真值
```

- Request body 加 `days: int = 7` (S-06a `CreateSessionReq` 只有 sid, 本轮加 days)

- S-06a 既有测试 (`test_create_ok`) 用 `seed_state=True` 默认行为, has_state=True 期望需更新断言

**S-06a 测试更新**: `test_create_ok` 现在 `has_state=True` (之前是 False). 旧 test 改 `assert resp.json()["has_state"] is True`. 测 `seed_state=False` 路径加一条 `test_create_no_seed_state` (caller 手工 mkdir 等价).

### 修订 B — swap 不调 `_impl_refine`, 改调 `recommend_meal` 然后 post-filter excludeIds (BLOCKER #2)

**问题**: iter 1 指出 `refine_session()` 不接 `excludeIds` 参数; `_impl_refine(refine_text="")` 会在 D-039 session 插一条空 intent refine 轮 (违反 CONTRACTS.md Faithful Refine).

**v2 决策**: swap 真实路径 = `recommend_meal(meal_type=..., root=ROOT)` 重跑 (新 D-039 session_id), 然后**手工 post-filter** candidates 内 `id in excludeIds` 的项. exclude_ids 是 best-effort (recommend_meal 可能返不到 5 条非 exclude 的; 不补全, 返剩余数量).

代价: 真实 swap 没有 refine_intent 加持 (用户没说 refine_text 时不该有 intent). intent 字段在 swap 的 Rec 里都是 "context-fit". 这是合理 UX.

### 修订 C — 提抽 `_impl_l1_refresh(root, force_no_llm, window_days)` (BLOCKER #3)

**问题**: iter 1 指出 plan 引用的 `_impl_l1_refresh / _run_l1_extract_internal` 在 web_api 不存在. 现有 L1 抽取逻辑分散在 `api_long_term_prefs_refresh` body + `_trigger_l1_extraction_async._worker` 闭包.

**v2 决策**: **本轮提抽出独立函数** `_impl_l1_refresh(root: Path, *, force_no_llm=False, window_days: int | None = None) -> dict`, 含原 `api_long_term_prefs_refresh` body 主体 (bootstrap or LLM branch + try/except). 然后:
- `api_long_term_prefs_refresh` 改成调 `_impl_l1_refresh(ROOT, force_no_llm=..., window_days=...)` 然后包 HTTPException 转换 (已有 RuntimeError/StoreCorruptError → HTTPException 转换)
- `_build_decision_async` BG task 内调 `_impl_l1_refresh(root, force_no_llm=False)` 真触发 L1 抽取

`§2 改` 加 `chisha/web_api.py: api_long_term_prefs_refresh 内部重构 (提抽 _impl_l1_refresh)`. 无对外行为变更 → 现有 `test_api_long_term_prefs_refresh*` 测试不破.

`_L1_EXTRACTION_LOCK` 语义 (iter 2 BLOCKER N2 修订): **BG task 必须持锁**. CONTRACTS.md:98 invariant 要求 reset/disable 抢 `_L1_EXTRACTION_LOCK` 防 worker 写盘中途清空 sandbox 目录污染 prod prefs. BG task 不持锁会让 reset 在 `extract_and_save` 中途介入.

**实现**: `_impl_l1_refresh(root, *, force_no_llm, window_days, acquire_lock: bool = True)` 内部:
```python
def _impl_l1_refresh(root, *, force_no_llm=False, window_days=None, acquire_lock=True):
    if acquire_lock:
        with _L1_EXTRACTION_LOCK:   # blocking acquire (BG task 等同 sid 内串行)
            return _do_l1_refresh(root, force_no_llm, window_days)
    return _do_l1_refresh(...)
```

`api_long_term_prefs_refresh` 既有 trylock fast-fail 路径保留 (用 acquire(blocking=False)) — 用户连按 refresh 不堆队列. BG task 调 `_impl_l1_refresh(...)` 默认 `acquire_lock=True` (blocking) — 单 sid 内 14 顿一个挨一个, contention 几乎无.

实际两条路径区别:
- HTTP `/api/long_term_prefs/refresh`: trylock + immediate fail with 409 if busy (现状, 不变)
- BG task in `_build_decision_async`: blocking acquire (保 CONTRACTS reset/disable 互斥)

### 修订 E — /recs /swap /refine 也加 done sentinel 检查 (BLOCKER #5)

**问题**: iter 1 指 /recs 在 `cur_idx == total_meals` 时仍会调 `meal_idx_to_slot(14) → (lunch, 8)` 进而走完整 recommend_meal LLM 调用.

**v2 决策**: **加 done guard helper**:

```python
def _ensure_not_done(sid: str | None, root: Path) -> dict:
    """读 state, 若 current_meal_idx >= total_meals raise HTTPException 409.
    返 state dict 供 caller reuse."""
    state_p = sandbox._state_path_for_sid(sid, root)
    if not state_p.exists():
        raise HTTPException(409, f"sandbox session {sid or '_default'!r} not initialized")
    try:
        s = json.loads(state_p.read_text(encoding="utf-8"))
    except (OSError, json.JSONDecodeError) as e:
        raise HTTPException(500, f"sandbox state corrupt: {e}")
    if not s.get("enabled"):
        raise HTTPException(409, "sandbox not enabled")
    cur = int(s.get("current_meal_idx", 0))
    total = int(s.get("total_meals", 14))
    if cur >= total:
        raise HTTPException(409, f"sandbox session done (meal_idx {cur}/{total})")
    return s
```

/recs, /eat, /swap, /refine 端点都先调 `_ensure_not_done(sid, ROOT)`, /skip 也加 (避免在 done 状态 skip 推进越界).

### 修订 F — atomic writes for all 4 落盘文件 (iter 1 Q9 升级)

**v2 决策**: `last_recs.json / history.json / meal_to_trace.json / decisions/{idx}.json` 都用 `tmp + os.replace` 原子写 helper. 单 helper `_atomic_write_json(path, data)`. 高 risk 任务里多花 20 行换强保证, 值.

### 修订 G — D-039 session TTL 与 sandbox 虚拟时间 (non-blocking #6 文档化)

iter 1 提 D-039 session 24h TTL 与 sandbox 虚拟时间 (`clock.today(root)` 在 sandbox 启用时是虚拟 today) 可能不匹配. v2 不在本任务修, 但在 §10 Q11 文档化: 单次 14 顿 session 全程在同一虚拟日历窗口内 (7 天虚拟时间) 不应触发 24h TTL (real-time wall clock 24h, session 创建到 done 通常 <1h). 真实 LLM 路径若超 1 天 caller 必须重 recommend. 测试 mock 路径无 D-039 session 写入, 完全 bypass.

## §3 API contract — 6 endpoints

所有端点 `_require_localhost`. 路径 `/api/sandbox/sessions/{sid}/...`. sid 校验复用 S-06a `_with_sandbox_sid` dep 模式 (path param 进 dep), 桶必须存在.

**所有端点都注入 `_with_sandbox_sid` 兼容 query 但路径 sid 是主**: 因为路径 `/sandbox/sessions/{sid}/eat` 已带 sid, 不需要 header. 但 dep 内部读 `request.query_params.get("session_id")` 不会与 path `sid` 冲突 (不同变量名). 为避免歧义, **本轮端点不挂 `_with_sandbox_sid` dep**, 而是 handler 内显式调 `_validate_and_route_sid(sid)` 校验 + `_sandbox_ctx(sid)` 包.

### 3.1 helper: `_validate_and_route_sid(sid: str) -> str | None`
- `sid == "_default"` → 返 None (走默认桶)
- 非 default: 校验 sid 格式 (`_validate_sid`); 校验 `sandbox.is_enabled(ROOT)` (409); 校验桶存在 (404); 返 sid
- 与 `_with_sandbox_sid` 等价语义, 用于 path sid 场景

### 3.2 `POST /api/sandbox/sessions/{sid}/recs?mock_recommend=0|1`

- Request: `?mock_recommend=1` (optional), body `{meal_type?: "lunch"|"dinner"}` (optional; 默认从 state 派生)
- Response 200: `{currentRecs: Rec[], recommend_session_id: str, applied_refine: null, meal_idx: int}`
- 行为:
  - 校验 sid + sandbox enabled. **`_ensure_not_done(sid, ROOT)`** (v2 修订 E: 防 done sentinel 状态触发 LLM). 读 state.current_meal_idx → 派生 meal_type (lunch/dinner via `meal_idx_to_slot`)
  - mock_recommend=1: `currentRecs = _mock_recs(meal_type, day)`; recommend_session_id = `f"mock_{secrets.token_hex(4)}"`; 不调 recommend_meal
  - mock_recommend=0: `with _sandbox_ctx(sid): out = recommend_meal(meal_type=..., root=ROOT)`; `currentRecs = [format_v2_to_rec(c) for c in out["candidates"]]`; recommend_session_id = out["session_id"]
  - 落 `sessions/{sid}/last_recs.json`: `{"recommend_session_id": ..., "candidates": [原始 v2 candidate], "currentRecs": [Rec view], "applied_refine": null, "meal_idx": cur_idx}`
- 失败: 409 sandbox disabled, 404 sid 桶不存在, 422 mock=0 但 recommend_meal 抛

### 3.3 `POST /api/sandbox/sessions/{sid}/eat?mock_recommend=0|1`

- Request body: `{rec_rank: int}` (1-5)
- Response 200: `{job_id: str, status: "running", new_meal_idx: int, meal_idx_eaten: int}`
- 行为 (同步部分):
  1. 校验 sid + sandbox enabled + **`_ensure_not_done`** (v2 修订 E)
  2. 读 `sessions/{sid}/last_recs.json` (404 if missing — 调用方必须先 POST /recs)
  3. 找 `last_recs.candidates[rec_rank - 1]` (400 if out of range)
  4. `cur_idx = state.current_meal_idx`
  5. `meal_to_trace[cur_idx] = last_recs.recommend_session_id` (落盘)
  6. 调 `_impl_accept(AcceptReq(session_id=recommend_session_id, candidate_rank=rec_rank, candidate=picked))` (复用 web_api 内部, **mock 路径跳过这步** — 因为 mock recommend_session_id 不在 D-039 session store 内, accept 会 404; mock 路径只更新 sandbox 文件)
  7. history append `{idx: cur_idx, state: "eat", dish: picked["restaurant"]["name"], session_id: recommend_session_id, accepted_at: now_iso}` → 落 `sessions/{sid}/history.json`
  8. `sandbox.advance_meal(sid=sid, root=ROOT)` 推进时钟
  9. 删 `sessions/{sid}/last_recs.json` (下顿要重 POST /recs)
  10. 启 BackgroundTask `_build_decision_async(sid, meal_idx=cur_idx, picked_rec, history_after, job_id, mock=is_mock, root=ROOT)`. 用 `asyncio.create_task` 或 FastAPI `BackgroundTasks` (后者最佳, 走 starlette 的 task queue, sync 函数 → thread pool); job_id 提前生成入表
  11. 返 `{job_id, status: "running", new_meal_idx, meal_idx_eaten: cur_idx}`

- BackgroundTask `_build_decision_async(sid, meal_idx, picked_rec_view, history, job_id, mock, root)` (v2 修订 D 用 Rec view-model + 修订 C 提抽 _impl_l1_refresh):
  1. with _JOB_LOCK: _JOB_TABLE[job_id] = {"status": "running", "started_at": iso, "sid": sid, "meal_idx": meal_idx}
  2. try:
     **with `_sandbox_ctx(sid)` 包整个 body** (BG task 在 thread pool 跑, ContextVar 必须 BG 函数内重 set, 不能依赖 caller thread):
     a. mock=False: `prev_prefs = _load_prefs_safe(root)`; 调 `_impl_l1_refresh(root, force_no_llm=False, window_days=None)` (v2 修订 C 提抽); `new_prefs = _load_prefs_safe(root)`
     b. mock=True: `prev_prefs = new_prefs = {}` (跳 LLM)
     c. `decision = build_decision(sid=sid, meal_idx=meal_idx, picked_rec=picked_rec_view, prev_long_term_prefs=prev_prefs, new_long_term_prefs=new_prefs, history=history, root=root)` — picked_rec_view 是 Rec 视图, 顶层 name/dishes/rank/l3 (v2 修订 D, 防 KeyError on v2 candidate restaurant.name)
     d. 原子写 `sessions/{sid}/decisions/{meal_idx}.json` (用 `_atomic_write_json`)
     e. with _JOB_LOCK: _JOB_TABLE[job_id] = {"status": "done", "result": {"decision": decision, "meal_idx": meal_idx}, "ended_at": iso}
  3. except Exception as e: with _JOB_LOCK: _JOB_TABLE[job_id] = {"status": "failed", "error": f"{type(e).__name__}: {e}", "ended_at": iso}; **不**写 decisions/{idx}.json (前端 polling 看 failed 状态不渲 D panel)

- 失败:
  - 409 sandbox disabled, 404 sid 桶, 404 last_recs.json 缺
  - 400 rec_rank 越界
  - 422 _impl_accept 抛 (非 mock 路径)
  - 500 advance_meal 抛

### 3.4 `POST /api/sandbox/sessions/{sid}/skip`

- Request body: `{reason?: str}` (校验 reason 在 `_VALID_SKIP_REASONS` 集合内)
- Response 200: `{new_meal_idx: int, meal_idx_skipped: int, decision: Decision}`
- 行为:
  1. 校验 sid + sandbox enabled + **`_ensure_not_done`** (v2 修订 E)
  2. `cur_idx = state.current_meal_idx`
  3. history append `{idx: cur_idx, state: "skip", reason, session_id: null}` → 落
  4. `sandbox.advance_meal(sid=sid)` 推进
  5. 删 last_recs (best-effort, 可能不存在)
  6. **同步**调 `build_decision(picked_rec=None, prev_prefs=None, new_prefs=None, history=history_after)` → 写 `sessions/{sid}/decisions/{cur_idx}.json`
  7. 返 `{new_meal_idx, meal_idx_skipped: cur_idx, decision}`

- skip 不启 BackgroundTask (无 L1 抽取需要触发)

### 3.5 `POST /api/sandbox/sessions/{sid}/swap?mock_recommend=0|1`

- Request body: `{exclude_ids?: list[str]}` (前端传当前 5 张 + 用户拒绝的)
- Response 200: `{currentRecs: Rec[], recommend_session_id: str}`
- 行为 (v2 修订 B: **不调** refine; 改调 recommend_meal post-filter):
  - 校验 sid + sandbox enabled + `_ensure_not_done`
  - 读 last_recs.json (404 if missing)
  - mock: `_mock_recs` 过滤 exclude_ids (返剩余 ≤ 5 条; 简化, 不补全)
  - 真实路径: `with _sandbox_ctx(sid): out = recommend_meal(meal_type=..., root=ROOT)`. 然后 `out["candidates"]` 过滤 `c["id"] in exclude_ids` 的项. 返剩余 (best-effort, 可能 < 5). 不调 refine_session, 不写 D-039 refine round, 不违反 Faithful Refine 契约.
  - 真实 swap = 新 D-039 session_id (recommend_meal 内部 `_gen_session_id`); 覆盖 last_recs.recommend_session_id
  - **不推进** meal_idx; 不动 history
  - currentRecs 走 adapter (intent="context-fit", 不带 refine_intent)

- 失败: 409 (done / disabled), 404 (sid 桶 / last_recs.json 缺)

### 3.6 `POST /api/sandbox/sessions/{sid}/refine?mock_recommend=0|1`

- Request body: `{text: str}`
- Response 200: `{currentRecs: Rec[], activeRules: {refine: [{label, sinceRound, sessionId}], blacklist: []}, recommend_session_id: str}`
- 行为:
  - 校验 sid + sandbox enabled + **`_ensure_not_done`** (v2 修订 E)
  - 读 last_recs (404 if missing)
  - mock: `_mock_refine(text)` 返 5 条 + label = text[:20]
  - 真实路径: `_impl_refine(RefineReq(session_id=last_recs.recommend_session_id, refine_text=text, excludeIds=[]))` (复用 web_api 内部)
  - 覆盖 last_recs.json: `applied_refine = {"label": text[:20], "sinceRound": <new round>, "sessionId": recommend_session_id}`; currentRecs adapter 转换
  - `activeRules.refine = [last_recs.applied_refine]` (单条); `blacklist = []` (S-06c 不接入 feedback_store.stopped)
  - **不推进** meal_idx; advance_meal 后 (下次 /recs / /eat) 会清空 last_recs → activeRules.refine 自动消失

- 失败: 409/404, 422 refine_session 抛

### 3.7 `GET /api/sandbox/sessions/{sid}/jobs/{job_id}`

- Response 200: `{status: "running"|"done"|"failed", error?: str, result?: dict, started_at, ended_at?}`
- 404: job_id 不存在
- in-memory 表, server restart 后丢; 前端容忍

## §4 Adapter `chisha/sandbox_adapter.py`

```python
def format_v2_to_rec(
    candidate: dict,                    # format_v2_candidate 输出
    *,
    l1_hits: list[str] | None = None,   # 从 L1 trace 提 top-2 token (调用方提供; mock 路径直接传)
    refine_intent: dict | None = None,  # refine 路径才有
    hard_filter_events: list[dict] | None = None,  # 派生 conflict
) -> dict:
    """v2 candidate → Rec view-model (前端 apps/sandbox-lab/src/types/sandbox.ts).

    必输入字段映射 (来自 format_v2_candidate):
      rank, id, restaurant.{name, distance_m, eta_min}, dishes[].canonical_name,
      total_price, score (L2 0-1), is_explore, reason_one_line, estimated_total_protein_g,
      estimated_total_oil

    派生字段:
      l3 = round(fit_score * 100) if fit_score else None
      boost = (l3 or 0) - round(l2 * 100)
      intent = refine_intent.summary_text if refine_intent else "context-fit"
      l1Hits = l1_hits or []  (top-2 token)
      conflict = 派生 hard_filter_events 内命中本 rec 的事件 (refine exclude_keywords 命中); 否则 null
      meta = {eta, dist, protein, oil}
    """
```

## §8 Regression risks

| 风险 | 严重度 | 缓解 |
|---|---|---|
| BackgroundTask 在 anyio worker thread 跑, _sandbox_ctx ContextVar 跨 thread 不 propagate | 高 | FastAPI `BackgroundTasks` 用同进程 thread pool, ContextVar **不**跨 thread. 在 BackgroundTask 函数内**显式重 set ContextVar**: 函数签名传 sid, 函数体内 `with _sandbox_ctx(sid): ...`. 测试 `test_eat_concurrent_sids_isolated` 防退化 |
| _impl_accept 在 mock 路径会 404 (mock_session_id 不在 D-039 store) | 高 | mock 路径**跳过** _impl_accept (eat 端点参数 mock=True 时直接 skip). 测试覆盖 |
| meal_to_trace.json 写盘失败 (并发 race) | 中 | json 落盘用 `tmp + os.replace` 原子写; 单 process 单用户场景, 不加锁 (FastAPI sync 端点串行) |
| history.json append race | 中 | 同上 atomic write + read-modify-write 不加锁 (S-06c 不上锁, gotcha 文档化, S-07 补) |
| advance_meal 在 14 顿后 raise → eat 端点返 500 | 低 | done sentinel: eat 端点先检查 cur_idx == total_meals → 409 "session done" (不进 advance_meal) |
| baseline_l2_snapshot drift | 极低 | 零打分链路改 |
| BackgroundTask 在 pytest TestClient 下不跑完 (异步) | 中 | TestClient `with` block 退出时会 join BG tasks. 测试用 `with TestClient(app) as c:` 模式 + 端点返后 poll job 表; 或测试改用 `BackgroundTasks().add_task(fn, ...)` 后立即跑同步 (FastAPI 测试模式行为) |
| L1 抽取真跑 (非 mock 路径) 慢 + 烧 token | 高 | 本轮**只测 mock 路径**, 真实路径靠 S-08 联调时手测 (mock_recommend=0). 测试不跑 LLM |

- (Q5) decision diff 在 skip 路径用空 prefs — 是, build_decision picked_rec=None 返"跳过未学习" diff=[], 与 prefs 无关
