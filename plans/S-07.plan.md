## A. Per-sid op lock (iter 2 Codex #2)

```python
_SESSION_OP_LOCK_REGISTRY: dict[str, _threading.Lock] = {}
_REGISTRY_LOCK = _threading.Lock()

@contextmanager
def _session_op_lock(routed_sid, *, timeout=30.0, action="op"):
    sid_key = routed_sid or _DEFAULT_SID_C
    with _REGISTRY_LOCK:
        lock = _SESSION_OP_LOCK_REGISTRY.setdefault(sid_key, _threading.Lock())
    ok = lock.acquire() if timeout is None else lock.acquire(timeout=timeout)
    if not ok: raise HTTPException(409, f"{action} blocked: sid={sid_key} busy")
    try: yield
    finally: lock.release()
```

Handler (eat/skip/swap/refine/rollback/branch) 入口 `with _session_op_lock(routed_sid, action=...):` timeout=30. `_build_decision_async` BG body 全程 `with _session_op_lock(sid, timeout=None)`. Branch handler 持 src + new 两把锁 (src 外, new 内).

## B. Rollback impl 全事务 (iter 2 Codex #1)

`actions: list[(target, new|None, kind)]`, kind ∈ {replace, delete, delete_dir}. 顺序 (state.json 最后):

1. meal_to_trace.json (replace)
2. history.json (replace)
3. long_term_prefs.json (replace, sandbox path via `data_root` + `_sandbox_ctx`)
4. meal_log.jsonl (replace, sandbox path)
5. recommend_log.jsonl (replace, sandbox path)
6. last_recs.json (delete)
7. decisions/{i>=meal_idx}.json (delete 多个)
8. recommend_trace/{tsid}.json (delete 多个 v2)
9. recommend_trace/{tsid}/ (delete_dir 多个 v3)
10. **state.json (replace) ← commit barrier**

stage `new_dir = bucket/.rollback_tmp/new/` 写所有新版本; `backup_dir = bucket/.rollback_tmp/backup/`.

执行循环:
- target 存在 → `os.replace(target, backup_dir/f_{idx}_{name})` (dir 用 `shutil.move`)
- replace kind: `os.replace(new_path, target)`
- delete kind: 备份完即"已删"

任一 raise → 反序还原: target 现存 unlink/rmtree, 然后 `os.replace(backup_p, target)`. finally `shutil.rmtree(.rollback_tmp)`.

state.json 必须列尾 — 任一 earlier failure → state.json 未动 + 其他 restored.

## D. FullSnapshot 契约 (iter 2 Codex #3 + #4)

新 Pydantic:

```python
class SessionMeta(BaseModel):
    sid: str; name: str = ""; days: int; seed: int = 0
    profile: str = "profile@v2"; origin: str = "blank"
    status: str  # running|done
    lastUsed: str = ""; currentMealIdx: int; totalMeals: int
    branchFrom: str | None = None

class Clock(BaseModel):
    idx: int; day: int; slot: str  # lunch|dinner
    total: int

class FullSnapshotResp(BaseModel):
    meta: SessionMeta; clock: Clock
    history: list[dict]; currentRecs: list[dict]
    lastDecision: dict | None
    activeRules: dict
    taste: list[dict] = []; keywords: list[dict] = []
    recent: list[str] = []; fatigue: list[dict] = []
```

`_build_full_snapshot(routed_sid, root)`:
- state.json → meta + clock (`meal_idx_to_slot` 派 slot, status=done 若 cur>=total)

- `decisions/{cur-1}.json` → lastDecision (None if cur=0 or 文件不存在)
- taste/keywords/recent/fatigue 空 list (S-08 填)

新 `GET /api/sandbox/sessions/{sid}` 端点返此. 不加 op lock (GET 只读). rollback handler 也返此. branch handler 仍返 `SandboxSessionMeta` (列表 view), 客户端拿到后另调 GET 拿 FullSnapshot.

## E. Branch

1. validate `0 <= from_meal_idx < src.current_meal_idx` (default src reject)
2. `with _session_op_lock(src_sid, action="branch"):`
   3. new_sid = `sandbox_{ts}_{secrets.token_hex(4)}`
   4. `shutil.copytree(src_bucket, new_bucket, ignore=shutil.ignore_patterns(".rollback_tmp"))` (iter 1 #5)
   5. `with _session_op_lock(new_sid, action="branch-new"):`
      6. `_rollback_session_impl(new_sid, from_meal_idx, ROOT, _internal=True)` (不 acquire lock, caller 已持)
      7. 读 new state, 加 `name`, `branch_from=src_sid`, `branch_from_meal_idx`, `started_at_real=now`, `sid=new_sid` → `_atomic_write_json`
8. 返 `SandboxSessionMeta(...)`

`_rollback_session_impl` 加 `_internal: bool = False` kw 控制是否 acquire lock (branch 走 True 跳过, 自己 handler 走 False 入口 wrap)
