### Step 3 · trace_store.py: 向后兼容读（**两条读路径都要改**）

**Codex audit 找到的盲点**: `read_trace` + `list_traces` 各自独立做 `__version` 严格匹配（line 139-142 + line 186-189）。**只改 read_trace 不够**，否则 v=1 trace 进不了 Sidebar 列表（`/api/debug/sessions` 走 `list_traces`）。

改造:
```python
ACCEPTED_TRACE_VERSIONS = {1, 2}

def _normalize_to_current_version(data: dict) -> dict:
    """v=1 → v=2 on-read migration. 只 inject 空 hard_filter_events 数组。
    不修改 __version 字段 (保留磁盘记录), 不写回磁盘。
    """
    version = data.get("__version")
    if version == 1:
        l1 = data.setdefault("l1", {})
        l1.setdefault("hard_filter_events", [])
    return data

def read_trace(...):
    ...
    version = data.get("__version")
    if version not in ACCEPTED_TRACE_VERSIONS:
        raise TraceVersionMismatch(...)
    return _normalize_to_current_version(data)

def list_traces(...):
    ...
    version = data.get("__version")
    if version not in ACCEPTED_TRACE_VERSIONS:
        continue  # 不算 corrupt, 静默跳过 (现有语义保留)
    # list 路径不需要注 hard_filter_events (Sidebar 只读 top1_summary 等元信息)
    ...
```

**关键**: 不写回磁盘。下一次 `write_trace`（含 refine merge `web_api.py:184-189`）自动落 v=2。

## Regression risks

| 风险 | 缓解 |
|---|---|
| **bump 到 v=2 后旧 trace 不可读** | `read_trace` + `list_traces` 都接受 {1, 2}，v=1 自动 migrate on read 注空数组 |
| **`list_traces` 漏改 → Sidebar 看不到 v=1 trace** | Codex audit 暴露,Step 3 已修 |
| **D-079 golden fixture 锁 __version=1** | Step 5.5 同步更新 fixture line 104 (1→2) + 加 hard_filter_events: [] |
| 自动 migrate 写回磁盘污染 What-if 冻结 | **不写回**,只读时注。下次 write_trace 自动落 v=2;refine merge (`web_api.py:184-189`) 把 v=1 trace 升 v=2 是预期行为 |
| dataclass 加字段破坏 asdict 调用方 | `schema_version` 有默认值，旧调用方零侵入 |
| `is_empty()` 误把 schema_version 算进去 | Codex audit 提示, 已确认现状只检查语义维度,不动 `is_empty()` |
| trace_store TypedDict 引入循环依赖 | TypedDict 是纯 type 提示，运行时是 dict，无依赖问题 |
| What-if response trace 漏 hard_filter_events | `_build_what_if_trace` 拷贝 base L1,migrated 字段会自动跟过去 |
