# S-05 Plan Review (Codex re-audit v5.2 — APPROVED)

Scope audited: only narrowed S-05 §1: static path layout, migration behavior, default-compat guard tests, and baseline comparison. Rollback, inspect, runtime web routing, cross-process locking, and `_meta.lock` reset timing remain deferred to S-06a.

## Result

The v5.2 plan addresses Codex iter 2's in-scope BLOCKED issue.

`relocated_legacy` is no longer based on `bool(relocated)`. The plan derives `relocate_satisfied` from per-artifact state:

- source absent: settled, covering already-moved and never-existing artifacts;
- source present plus destination present: interrupted, not satisfied;
- newly copied and renamed artifacts: satisfied after source is moved aside.

Both meta-write branches now use this derived state:

- relocate-only branch: `relocated_legacy = relocate_satisfied or already_relocated`
- first migration branch: `relocated_legacy = bool(relocate_legacy and relocate_satisfied)`

The three new v5.2 tests cover the blocked cases:

- `test_relocate_twice_second_is_already_migrated`
- `test_relocate_with_no_artifacts_marks_satisfied`
- `test_relocate_interrupted_mid_run_not_satisfied`

## Codebase Cross-Check

Current `data_root` matches the plan assumptions: sandbox disabled routes to prod; default sid routes flat to `logs/sandbox/<rel>`; non-default sid routes to `logs/sandbox/sessions/{sid}/<rel>`.

Current `sandbox_context` reserves `_legacy`, so the migration target cannot collide with a valid user session id.

Current `trace_store` routes through `data_root.recommend_trace_dir(root)`, so no direct `trace_store` change is needed for this narrow scope.

The baseline comparison command now explicitly passes `--before-dir` and `--after-dir`, matching `scripts/compare_traces.py` and avoiding the previous false-positive risk.

## Non-Blocking Notes

- `plans/S-05.plan.md` line 60 still says the CLI relocates `sessions/`, but the implementation block and `test_sessions_dir_not_relocated` correctly exclude it. This is stale prose, not a blocker.
- The relocate-only branch writes `relocated_at` even for an interrupted state where `relocated_legacy` remains false. That is noisy metadata, but it does not break idempotency or default compatibility in this scope.
