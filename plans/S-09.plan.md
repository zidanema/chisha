### 6. debug-ui 接 URL query (Iter 1 修订)

**Codex Iter 1 #1 接受**: useWaTrace 的 bootstrap (useWaTrace.ts:178-208) 在 mount 时 fetch `/api/traces`, 当 backend list 不含 mock 默认 activeTraceId 时调 `setActiveTraceIdState(list[0].id)` 会覆盖 URL 选中的 trace. **mount-once useEffect 在 App 顶层无法稳赢这场竞速**.

修订方案: 把 URL parsing 下推到 useWaTrace 内部, 跟 bootstrap 同 effect 决定 initial activeTraceId.

`apps/debug-ui/src/hooks/useWaTrace.ts`:
- 顶部加 helper:
  ```ts
  function readTraceFromUrl(): string | null {
    if (typeof window === "undefined") return null;
    try {
      const p = new URLSearchParams(window.location.search);
      const t = p.get("trace");
      return t && t.length > 0 ? t : null;
    } catch { return null; }
  }
  ```
- `useState<string>(ACTIVE_WA_TRACE.meta.id)` 改成 `useState<string>(() => readTraceFromUrl() ?? ACTIVE_WA_TRACE.meta.id)`.
- bootstrap effect 内 `if (!list.find((t) => t.id === activeTraceId)) setActiveTraceIdState(list[0].id)` 改 (Iter 2 修订):
  ```ts
  const urlTrace = readTraceFromUrl();
  if (urlTrace) {
    // URL 指定 → 不动 activeTraceId, 不管它是否在 list 里. fetchTraceDetail
    // 会决定 success / 404 (落 mock fallback toast).
  } else if (!list.find((t) => t.id === activeTraceId)) {
    setActiveTraceIdState(list[0].id);
  }
  ```
- 关键: 只有"无 URL trace 请求"时才允许 fallback 到 list[0]. URL 指定一律 pin 住, 让 `/api/trace/{tsid}` 直查 (web_api.py:2766) authoritative.
- 不存在的 trace id: detail fetch 走 404 → 现有 catch 已 fallback mock + toast (useWaTrace.ts:230-238). 用户看到的是 mock trace + toast 提示 "trace 读失败", 这符合 spec §4 "如果 trace 不存在 → 仍然走 mock fallback".

`apps/debug-ui/src/App.tsx`:
- 仍加一个 mount-once useEffect 但**只**负责 `setActiveRound` (默认 R1, URL 可覆盖):
  ```ts
  useEffect(() => {
    const params = new URLSearchParams(window.location.search);
    const round = params.get("round");
    if (round) setActiveRound(round);
  }, []);  // eslint-disable-line react-hooks/exhaustive-deps
  ```
- 注: 已有 effect (App.tsx:86-93 `setActiveRound((prev) => ...rounds.find(prev) ? prev : latest)`) 会在 rounds 到达后用 `rounds.find` 检查, 找不到时 fallback latest. 如果 URL round=R1 而 trace 含 R1 → 保留 R1; 否则退化 latest. 行为符合 spec ("如果 trace 不存在 → 仍然走 mock fallback").

- 不动 URL (不 replaceState, 不 push) → 满足 CONTRACTS line 110.
- 不破坏 localStorage `chisha:tbCollapsed` / `chisha:theme`.

测试: chrome-devtools-mcp e2e 步骤要 navigate `/?trace=<非 list[0] 的 tsid>&round=R1` 并验证 trace 仍是该 tsid (不被 list[0] 覆盖).
