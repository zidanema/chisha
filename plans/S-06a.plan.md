### 修订 G — non-default sid 运行时路由必加 `sandbox.is_enabled(ROOT)` gate

**问题**: iter 3 指出 v3 `_with_sandbox_sid` dep 只校验桶目录存在, 不校验 `sandbox.is_enabled(ROOT)`. failure mode:

1. `POST /api/sandbox/init` → 写 state.json (enabled=True) + `_meta.json`
2. `POST /api/sandbox/sessions` body `{"sid":"s1"}` → mkdir `logs/sandbox/sessions/s1/`
3. `POST /api/sandbox/disable` → 改 state.json (enabled=False), **不**删 `_meta.json`, **不**删 `s1/` 桶目录 (符合现状: disable 只切 flag)
4. `GET /api/recommend` with `X-Session-Id: s1`
   - dep: 桶目录 `s1/` 存在 → pass → `set_sandbox_session("s1")` → ctx sid=s1
   - handler 调 `recommend_meal(root=ROOT)` (v3 修订 D)
   - `data_root.recommend_log_path(ROOT)` → `_maybe_sandbox(ROOT, session_id=None)` → `_resolve_sid(None)` 回退到 ctx sid="s1" → 看 `is_enabled(ROOT) == False` → **fall back 到 prod path** `logs/recommend_log.jsonl` (data_root.py:62-93 现有逻辑)
   - 结果: ctx sid 被静默忽略, 写到 prod, 用户预期 "写到 s1 桶" 落空, 且**没有任何 4xx 提示**.

**v4 决策**: `_with_sandbox_sid` dep 在接收到非 default sid 时, **额外加 `sandbox.is_enabled(ROOT)` 检查**. disabled 时直接 4xx, 不进 handler.

具体改 §4.2 dependency 流程:

```python
def _with_sandbox_sid(request: Request, session_id: str | None = Query(None)):
    sid = request.headers.get("X-Session-Id") or session_id
    if sid is None or sid == "_default":
        yield None
        return
    # 1. sid 格式校验
    try:
        _validate_sid(sid)
    except ValueError as e:
        raise HTTPException(400, f"invalid session_id: {e}")
    # 2. v4 新增: sandbox 必须 enabled 才能跑非 default sid (防 silent fall back)
    from chisha import sandbox as _sandbox
    if not _sandbox.is_enabled(ROOT):
        raise HTTPException(
            409,
            f"sandbox layout disabled; cannot route session_id={sid!r}. "
            "POST /api/sandbox/init or POST /api/sandbox/<advance|reset> first.",
        )
    # 3. 桶存在校验
    if not (ROOT / "logs" / "sandbox" / "sessions" / sid).is_dir():
        raise HTTPException(404, f"unknown sandbox session_id={sid!r}")
    # 4. 注入 ctx
    with set_sandbox_session(sid):
        yield sid
```

返回码选 **409 Conflict** (state-related disagreement; "桶存在但 layout disabled" 是状态冲突, 不是 client request 格式错), 与 §3.4 rename collision 409 一致风格. 若 reviewer 觉得 400 更合适本节可调, 但默认 409.

**trade-off**:
- (+) 修补 iter 3 cited silent fall back; 用户拿到清晰 4xx 反馈, 不会写脏 prod
- (+) 与 `data_root._maybe_sandbox` 在 disabled 时回退的现有契约**不冲突** — dep 层先拦, 拦不住时 data_root 兜底 (双层防御)
- (+) `_default` / 不传 sid 路径完全不变, 不破坏 default 桶行为 (即使 disabled 也能跑 prod 路径)
- (-) 用户 init → create s1 → disable 后, 想再 enable 跑 s1 必须先 advance / reset 重新打开 sandbox — 但这是合理 UX (disabled 状态就不该路由到 s1)
- (-) dep 多调一次 `is_enabled` (读 state.json) → 每请求 +1 disk read; 可忽略 (state.json 本地 LRU 由 fs cache 兜)

**为什么不在 data_root 层加这检查**: data_root.py 是路径决策层, 它不该返回 HTTPException; "拒掉请求" 是 web 层职责. 保持分层.

### 修订 A — 路径冲突: 保留 `logs/sandbox/sessions/{sid}/` 但加 mixed-namespace discriminator

**问题**: iter 1 担心 `logs/sandbox/sessions/{sid}/` (CRUD 桶) 与 `logs/sandbox/sessions/*.json` (D-039 default-bucket recommend-session 文件, `chisha/session.py:62`) 串内容; `data_root.sessions_dir()` 也把 `logs/sandbox/sessions/` (sid=`_default` 分支, line 114) 当 D-039 目录.

- Sandbox 非 default 桶: **子目录** `logs/sandbox/sessions/{sid}/` (sid 符合 `_validate_sid` regex, **排除** `_legacy` reserved).
- `list_sessions(root)` 实现:

- **trade-off**:
  - (+) 不改 S-04 已落实的 `data_root._maybe_sandbox` 契约 (line 93) → 零 path 链路风险
  - (+) 不动 S-05 migration `_LEGACY_ARTIFACTS` 清单 (line 37-48)

  - (-) `logs/sandbox/sessions/` 目录是 mixed namespace, 维护时需理解 "文件=default 桶 / 目录=非 default 桶" 约定 — 用 docstring + mixed-namespace 测试 (§6.2 新增 case) 防御

### 修订 B — `/sandbox/init` 不创建 `_meta.json`: S-06a 让 `sandbox.init()` 兼容 ensure

**问题**: iter 1 指出 plan v1 用 `sandbox.has_sandbox_meta(ROOT)` 作 layout 检查 (拒绝 create_session), 但 `sandbox.init()` (`chisha/sandbox.py:181`) 只写 `state.json`, 不写 `_meta.json`; `_meta.json` 仅由 `migrate_to_v2()` 产生, 而 `/api/sandbox/init` (`web_api.py:838`) 不调 migration. 结果 plan v1 流程 `init → POST /sessions` 仍会被 400 拒.

**v2 决策**: 选 (A) — **`sandbox.init()` 内部 ensure `_meta.json` 存在 (idempotent)**.

具体实现 (写入 `chisha/sandbox.py:init`, 在 `_atomic_write` state.json 后追加):

```python
# S-06a: ensure layout-v2 marker so post-init flows (POST /sessions, inspect
# three-state) see a coherent v2 layout. Idempotent: 若 _meta.json 已存在
# (migration 已跑过) 则不覆盖.
from chisha import sandbox_migration
if not sandbox_migration.read_meta(root or _project_root()):
    sandbox_migration._atomic_write_meta(
        root or _project_root(),
        {
            "schema_version": sandbox_migration.SCHEMA_VERSION,  # =2
            "created_by": "sandbox.init",
            "created_at": _now_real_iso(),
            "migrations": [],  # init 不算 migration
        },
    )
```

- **trade-off**:
  - (+) `init → POST /sessions` 单步可用, 不需要先调 migration
  - (+) `init → disable → inspect` 真返 `has_layout: True, sessions: [default]` (符合三态语义)
  - (+) 与现有 `migrate_to_v2()` 幂等: migration 跑了 init 不再覆盖, init 跑了 migration 检测 schema_version=2 自然 no-op
  - (-) `sandbox.init()` 多依赖 `sandbox_migration` 模块 — 已是同包, 加 import 即可, 无循环
  - (-) `_meta.json` 不再只是 "migration 跑过" 的证据, 也是 "init 跑过" 的证据 — 在 `created_by` 字段区分

**替代方案考虑过**: (B) 改契约要求 explicit migration 在 init 之后 — 否决理由是会给所有 caller (test setup + 文档示例 + frontend) 增一步 `POST /sandbox/migrate`, 而本任务无 `/sandbox/migrate` 端点 (S-05 migration 是 CLI/ad-hoc 跑, 不通过 web). 让 init 兼容产生 `_meta.json` 更自然.

## §3 API contract — 4 sessions CRUD endpoints

所有端点 `_require_localhost`. 路径前缀已是 `/api` (router prefix).

### 3.1 `GET /api/sandbox/sessions`

- Response 200: `SessionsListResp = { sessions: list[SandboxSessionMeta] }`
- `SandboxSessionMeta`:
  ```
  {
    "sid": str,                              # "_default" 或合法 sid
    "is_default": bool,                       # sid == "_default"
    "created_at": str | null,                 # ISO 时间; default 桶取 _meta.created_at / state.started_at_real
    "size_bytes": int,                        # 桶内文件总大小 (粗略, listing 不递归 stat 失败回 0)
    "has_state": bool,                        # state.json 是否存在 (default 桶 = logs/sandbox/state.json, 非 default 桶预留 False — S-06b 才填)
  }
  ```
- 派生规则 (v2 mixed-namespace discriminator):
  - default 桶: 永远出现, 即使 layout 未建 (此时 created_at=null, size_bytes=0, has_state=False)
  - 非 default 桶: 来自 `logs/sandbox/sessions/` **子目录** (跳 `.json` 文件 = D-039 default 桶, 跳 `_legacy` reserved, 跳 sid regex 不合法的目录名)
  - 排序: default 第一, 其余按 sid 字典序
- 失败:
  - 403 非 localhost (现有 `_require_localhost`)
  - 500 不抛 (`logs/sandbox/sessions` 不存在或 IO 错 → 仅返 default)

### 3.2 `POST /api/sandbox/sessions`

- Request body: `CreateSessionReq = { sid: str }`
- Response 201: 新 `SandboxSessionMeta`
- 失败:
  - 400 `sid == "_default"` (reserved)
  - 400 sid 格式不合法 (`_validate_sid` 抛 ValueError → wrap 400; 含 `_legacy` reserved / regex 不匹配 / 空 / 超长)
  - 400 `not sandbox.has_sandbox_meta(ROOT)` (layout 未建; v2 修订 B 后, `sandbox.init` 已自带 `_meta.json`, 此 400 主要兜底未 init 调用; hint: "先 POST /api/sandbox/init")
  - 409 桶已存在 (`logs/sandbox/sessions/{sid}/` 已有)
  - 403 非 localhost
- 行为: `(root / "logs" / "sandbox" / "sessions" / sid).mkdir(parents=True, exist_ok=False)` (exist_ok=False → atomic check); 不预写任何 state (S-06b 真造 state.json).

### 3.3 `DELETE /api/sandbox/sessions/{sid}`

- Path param: sid
- Response 200: `{ ok: True, deleted_sid: sid }`
- 失败:
  - 400 `sid == "_default"` (forbid; "default 桶禁删")
  - 400 sid 格式不合法 (`_validate_sid` 抛 → 400)
  - 404 桶不存在 (idempotency 选择: 拒 404 — 与 GET single 一致, 避免幂等遮蔽 bug)
  - 403 非 localhost
- 行为: `shutil.rmtree(logs/sandbox/sessions/{sid}/)`. 不做 ctx 检查 (即便正巧某请求带这 sid 进来读, GET 端点会先校验存在 → 404).

### 3.4 `POST /api/sandbox/sessions/{sid}/rename`

- Path param: 老 sid
- Request body: `RenameSessionReq = { new_sid: str }`
- Response 200: 新 `SandboxSessionMeta` (sid=new_sid)
- 失败:
  - 400 `sid == "_default"` OR `new_sid == "_default"` (forbid)
  - 400 sid / new_sid 格式不合法
  - 400 `sid == new_sid` (no-op 显式拒, 不静默 200)
  - 404 老桶不存在
  - 409 新桶名已被占用 (collision)
  - 403 非 localhost
- 原子性: `os.rename(old_dir, new_dir)` — 同 fs 下原子. **不**用 `shutil.move` (跨 fs 时退化为 copy+rm 半态). 若 `os.rename` 抛 `OSError(EXDEV)` (跨 fs) → 500 with explicit msg "rename requires same fs; manual intervention needed". 不静默 fallback 到 copy+rm.

## §4 non-default sid runtime routing in web_api

### 4.1 sid 来源 (header > query > 缺省)

- `X-Session-Id` HTTP header
- `?session_id=` query param (FastAPI Depends 提供 fallback, 让 frontend 能用纯 GET URL 验证)
- 都没传 → `None` → 不进 `set_sandbox_session` → ctx 内 `current_sandbox_session()` 返 None → `data_root._resolve_sid` 走 `_default` → 与今天行为完全一致 (D-077 flat).

### 4.2 注入点

新增 FastAPI dependency `_with_sandbox_sid(request)` (generator):
1. 取 header / query 候选, header 优先
2. None / `_default` → 直接 yield None (ctx 不进 — 跟 default 桶行为一致, 避免对老测试有副作用)
3. 非 None: `_validate_sid(sid)` (失败 → 400 with explicit msg)
4. **(v4 修订 G)** `sandbox.is_enabled(ROOT) == False` → 409 `sandbox layout disabled; cannot route session_id={sid!r}` (**fail-loud, 防 data_root silent fall back 到 prod**)
5. 检查桶存在: `(ROOT / "logs" / "sandbox" / "sessions" / sid).is_dir()` False → 404 `unknown sandbox session_id={sid!r}` (**fail-loud, 不 silent fall back to default 桶**)
6. 桶存在 → 用 contextmanager wrap: `with set_sandbox_session(sid): yield`

注入方式: FastAPI `Depends(_with_sandbox_sid)`, generator function `yield` 在 with 内. 端点签名加 `_=Depends(_with_sandbox_sid)`. 因为 dependency 是 sync, ContextVar 在请求 thread 内有效, 端点 handler 调 data_root/clock 自然取到 sid (S-04 ContextVar 已支持).

### 4.3 应用范围 (本轮挂哪些端点)

挂在以下读写端点 (所有走 data_root / clock 的端点):
- `GET /api/recommend`, `POST /api/refine`, `POST /api/accept`, `POST /api/skip`
- `GET /api/profile`, `PUT/POST /api/profile`
- `GET /api/history`
- `POST /api/feedback`, `GET /api/feedback/inbox`, `GET /api/feedback/recent`, snooze/stop, `GET /api/feedback/{sid}` (注: 这里 sid 是 D-039 recommend session 别名, 与 sandbox sid 概念区分; dependency 注的是 `X-Session-Id`, 不冲突)
- `POST /api/long_term_prefs/refresh`
- `GET /api/sandbox/inspect`, `GET /api/sandbox/state` (state 读不同 sid 的 state.json — S-06b 真实现; 本轮 inject 但 `sandbox.state` 仍 ignore sid, 行为不变, 不破坏现状)

**不挂** (sandbox 生命周期管理本身, 不应被 sid 影响):
- `/api/sandbox/init`, `/advance`, `/reset`, `/disable` — destructive global (S-04 `_reject_nondefault_sid` 防护; 即使 frontend 误传 X-Session-Id, dependency 422 之前会先在 sandbox 模块层 raise)
- `/api/sandbox/sessions*` (CRUD 自己) — sid 在 path param 里
- `/api/debug/*` 和 `/api/traces*` — 这些是 trace 浏览, 不受当前 sandbox sid 影响 (trace 文件名自带)

### 4.4 fail-loud 保证

S-04 已在 `data_root._maybe_sandbox` 加了 "sandbox 关 + 显式非 default sid → RuntimeError" 防护. 本轮 web_api dependency 层先校验桶存在 → 4xx, 不会进到 data_root raise. 但**保留** S-04 兜底.

测试守门: 请求带不存在 sid → 404, 后端不写任何 prod 文件 (验 `data/long_term_prefs.json` mtime 不变).

## §5 `/sandbox/inspect` 三态

S-05 deferral A. 当前实现 (`web_api.py:1056-1118`) 只看 `_sandbox.is_enabled(ROOT)`:
- True → 返 full snapshot
- False → 返 `{enabled: False}` (无视 layout 是否已建)

S-06a 改成三态 (v2: 因 §0 修订 B 后 `init` 也写 `_meta.json`, 三态触发条件 coherent):

| 状态 | 触发条件 | 返回 |
|---|---|---|
| no-layout | `not sandbox.has_sandbox_meta(ROOT) and not sandbox.is_enabled(ROOT)` | `{enabled: False, has_layout: False, sessions: []}` |
| layout-disabled (新) | `sandbox.has_sandbox_meta(ROOT) and not sandbox.is_enabled(ROOT)` | `{enabled: False, has_layout: True, sessions: [list_sessions(ROOT)]}` — 至少含 default + 任何已创建非 default 桶 |
| layout-enabled | `sandbox.is_enabled(ROOT)` | 现状 (state + prefs + feedbacks + meal_log + accepted_count) + 新增 `has_layout: True` + `sessions: [list_sessions]` |

判定逻辑: 用 `sandbox.has_sandbox_meta` (S-05 已加) + `sandbox.is_enabled`. 不引入新 helper, 不动 path 决策链. v2 修订 B 后, `init` 自带 `_meta.json` → init→disable→inspect 真返 layout-disabled (而不是误回 no-layout).

## §7 Regression risks + rollback notes

### 风险矩阵

| 风险 | 严重度 | 缓解 |
|---|---|---|
| sid 注入到 history/feedback 端点后 default 桶请求行为变更 | 高 | dependency 在 sid=None / `_default` 时**直接 yield**, 不进 `set_sandbox_session`, ctx 内 `current_sandbox_session()` 仍 None → S-04 行为 0 变化 |
| 误把 destructive endpoint 也挂 sid routing | 高 | §4.3 显式不挂 init/advance/reset/disable; 测试守门 `test_destructive_endpoint_ignores_sid` |
| rename mv 半态 (跨 fs / 文件锁) | 高 | 用 `os.rename` 同 fs 原子; 跨 fs 直接 500 不静默 fallback; defer 跨 fs 处理到 S-07 |
| delete 误删默认 | 高 | 端点层 400; `sandbox.delete_session` 模块层亦校验 |
| inspect 三态 break frontend | 低 | 新字段 `has_layout`, `sessions` 只是**新增**, 旧字段 (enabled / state / long_term_prefs / ...) 不动 |
| baseline_l2_snapshot drift | 极低 | 零打分链路改动. 仍走 baseline 双跑确认 |
| **(v2 新增) `init` 写 `_meta.json` 与 migration 冲突** | 低 | `init` 用 `migration.read_meta(root)` 判存在再写, idempotent; migration 跑后 init 不覆盖 (因 `schema_version=2` 已在 `_meta.json` 内, `read_meta` 返非 None → init skip 写) |
| **(v2 新增) mixed namespace 维护成本** | 中 | docstring + `list_sessions` discriminator 严格 (跳 `.json`/`_legacy`/非法 sid 目录) + `test_list_mixed_namespace` 防退化 |

### Rollback

- 全部改动局限在 `chisha/web_api.py` + `chisha/sandbox.py` 加函数 (v2 也含 init 增 _meta.json) + 3 个新 test 文件. 直接 git revert HEAD 即恢复.
- 数据落盘改动: rename 在测试环境用 tmp_path; 生产用户跑 rename 前需有 layout v2 (已 migration 或已 init). 无 schema 变更, 无 trace_store v3 风险.
- v2 修订 B 的 `_meta.json` ensure: revert 不影响已存在的 `_meta.json` (init 不覆盖 existing).
